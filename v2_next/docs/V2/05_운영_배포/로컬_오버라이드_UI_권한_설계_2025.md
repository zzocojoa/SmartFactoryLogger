# 로컬 오버라이드 UI/권한 설계 (2단계)

## 1. 목적
- 중앙 설정을 기본값으로 유지하면서, **현장 예외를 안전하게 허용**한다.
- 오버라이드 남용을 방지하고 **권한 기반으로 통제**한다.

## 2. 기본 정책
- 기본값: 오버라이드 **OFF**
- 오버라이드 ON 시에만 로컬 변경 유지
- 중앙 `force=true` 배포 시에는 로컬 오버라이드 무시

## 3. UI 구성
### 3.1 설정 화면 상단 배너
- 배너 문구: “로컬 오버라이드 모드: ON/OFF”
- 상태 배지 색상:
  - OFF: 중립(회색)
  - ON: 경고(노랑)
- ON 상태는 항상 표시

### 3.2 토글 스위치
- 위치: 설정 모달 상단 또는 보안 섹션
- 라벨: “로컬 오버라이드 허용”
- 활성 시 확인 다이얼로그 표시
  - “중앙 설정과 충돌할 수 있습니다. 계속 진행하시겠습니까?”

### 3.3 변경 이력 표시
- 오버라이드 모드에서 변경된 항목은 “LOCAL” 태그 표시
- 변경 요약에 로컬 변경 여부 표시

## 4. 권한 정책
- Admin: 오버라이드 ON/OFF 가능
- Operator: 변경은 가능하나 오버라이드 ON/OFF는 불가
- Viewer: 읽기 전용

## 5. 인증 흐름
- 오버라이드 토글 시 비밀번호/인증 요구
- 인증 성공 시에만 토글 가능
- 인증 실패 시 변경 차단

## 6. 로컬 저장 방식
- `config.ini` 저장 시 `config_meta.json`에 아래 정보 기록
```json
{
  "override_enabled": true,
  "override_by": "operator01",
  "override_at": "2025-01-02T10:30:00Z"
}
```

## 7. UX 정책
- OFF 상태에서 로컬 변경 시도 → “중앙 설정 모드에서는 변경이 제한됩니다.”
- ON 상태에서 저장 시 → “로컬 오버라이드 적용됨” 배지 표시
- 중앙 강제 배포 시 → 오버라이드 자동 해제 + 경고 토스트

## 8. 수용 기준(AC)
- 오버라이드 OFF일 때 로컬 변경은 저장 불가
- Admin만 토글 가능
- 오버라이드 ON 시 변경 내역에 LOCAL 태그 표시
- 강제 배포 시 오버라이드 자동 해제
