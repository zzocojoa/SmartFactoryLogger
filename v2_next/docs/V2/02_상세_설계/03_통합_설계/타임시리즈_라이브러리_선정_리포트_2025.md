# 타임시리즈 라이브러리 선정 리포트 (알루미늄 압출 실시간 데이터)

## 1. 대상 데이터 범위
`v1_legacy/docs/reports/통신/전체_통신_주소_맵핑_현황.md`의 "## 2. 데이터 맵핑 테이블" 기준으로
다음 17개 신호를 타임시리즈에 표시한다.

- SPOT(HTTP): `Temperature`
- Extruder(TCP/Melsec): `Press`, `Temp_F`, `Temp_B`, `Speed`, `EndPos`, `Count`, `Billet`
- LS PLC(TCP/XGT): `Mold1`, `Mold2`, `Mold3`, `Mold4`, `Mold5`, `Mold6`, `Billet_Temp`, `At_Temp`, `At_Pre`

특성 요약:
- 데이터 소스가 3계통(Extruder/LS/Spot)으로 분리됨
- 샘플링 속도: Extruder/LS는 0.2s(5Hz) 기준, SPOT는 3s 기준
- 단위가 혼재(온도/압력/속도/카운트/위치) → 다중 축 필요
- 실시간 갱신 + 최근 구간(예: 10~30분) 슬라이딩 표시가 적합

## 2. 라이브러리 요구사항
- 실시간 스트리밍(초당 수십 포인트 이상) 성능
- 다중 Y축/단위 표시, 임계값 라인, 범례 필터
- 줌/팬/툴팁/크로스헤어 지원
- React 통합 용이성 + 현재 스택(@grafana/scenes)과의 호환성
- 장기 데이터는 다운샘플링/윈도잉 처리 가능해야 함

## 3. 후보 라이브러리 비교

### A) Grafana Scenes TimeSeries Panel (@grafana/scenes + @grafana/ui)
- 장점
  - 이미 v2에서 사용 중인 스택과 동일(의존성 추가 최소화)
  - 다중 축, 임계값, 색상 규칙, 범례/툴팁/크로스헤어 등 기능 완비
  - Grafana DataFrame 기반으로 실시간 스트림 설계가 용이
  - 운영/관측성 패널과 UI 일관성 유지
- 단점
  - 패키지 의존성이 무겁고 React 18 고정 이슈가 있음
  - Grafana-specific 구조에 맞춰 데이터 변환 필요

### B) Apache ECharts
- 장점
  - 성능/기능 균형 우수, 대규모 커뮤니티
  - 다중 축/데이터줌/임계값 표현 유연
  - React 래퍼 사용 시 통합 용이
- 단점
  - 현재 Grafana Scenes 기반 UI와 일관성 맞추려면 스타일/상태 관리 추가 작업 필요
  - 임계값/알림 규칙은 별도 구현 필요

### C) uPlot
- 장점
  - 매우 가볍고 빠름(실시간 스트림에 최적)
  - 낮은 CPU/메모리 사용
- 단점
  - 다중 축/범례/상호작용 기능이 기본은 제한적
  - UI/UX 완성도 확보를 위한 커스텀 개발량이 큼

### D) Highcharts / Plotly
- 장점: 기능 풍부
- 단점: 상용 라이선스 고려 또는 번들 크기 부담 큼

## 4. 권장안
### 1순위: Grafana Scenes TimeSeries Panel
현재 v2에서 이미 Grafana Scenes를 사용하고 있어, 동일 생태계 내에서 TimeSeries를 구성하는 것이
개발 리스크와 유지보수 비용을 최소화한다. 특히 임계값/범례/다중축/성능 최적화가 내장되어 있어
17개 신호를 안정적으로 표시하는 데 적합하다.

### 2순위(대안): Apache ECharts
Grafana 의존성을 줄이거나 독립적인 차트를 구성해야 한다면 ECharts가 현실적인 대안이다.
다만 기존 UI/설정 체계와의 일관성을 맞추기 위해 추가 설계가 필요하다.

## 5. 설계 제안 (가독성 관점)
- 17개 신호를 단일 그래프에 모두 표시하면 가독성이 급격히 저하됨
- 권장 분리 예시:
  - 공정 핵심(Press/Speed/EndPos/Count/Billet)
  - 온도군(Temperature/Temp_F/Temp_B/Billet_Temp/Mold1~6/At_Temp)
  - 환경/보조(At_Pre 등)
- 또는 단일 그래프 + 범례 토글 + 그룹 필터 제공

## 6. 결론
실시간 압출 공정 대시보드 특성(다중 신호, 고빈도, 임계값 표시, 운영 일관성)을 고려하면
현재 스택과 호환되는 Grafana Scenes TimeSeries를 기본 라이브러리로 채택하는 것이 최적이다.
추가로 독립성/경량화가 필요할 경우 ECharts를 대안으로 유지한다.
