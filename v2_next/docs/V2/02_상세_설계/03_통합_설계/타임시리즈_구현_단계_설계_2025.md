# 타임시리즈 구현 단계 설계 (Grafana Scenes 기반)

## 1. 목표
- 실시간 수집 데이터를 타임시리즈로 표시하여 공정 이상/변화를 즉시 파악
- 17개 신호를 다중 단위/다중 축으로 안정적으로 시각화
- 기존 Grafana Scenes 기반 UI와 일관성을 유지하면서 성능 리스크 최소화

## 2. 대상 데이터(17개 신호)
데이터 키는 `FactoryData` 기준으로 정규화한다.

- SPOT: `Temperature` → `Spot`
- Extruder: `Press`, `Temp_F`, `Temp_B`, `Speed`, `EndPos`, `Count`, `Billet`
- LS PLC: `Mold1`, `Mold2`, `Mold3`, `Mold4`, `Mold5`, `Mold6`, `Billet_Temp`, `At_Temp`, `At_Pre`

단위/축 분류(권장):
- 압력/속도/위치/카운트: `Press`, `Speed`, `EndPos`, `Count`, `Billet`
- 온도군: `Spot`, `Temp_F`, `Temp_B`, `Billet_Temp`, `Mold1~6`, `At_Temp`
- 환경: `At_Pre`

## 3. 구현 단계(상세)

### 1) 시계열 카탈로그 정의
- 시리즈 메타 정의 배열 생성(키, 라벨, 단위, 축 그룹, 기본 가시성).
- 예: `{ key: "Press", label: "메인압력", unit: "pressure", axis: "process", visible: true }`
- 온도군/공정군/환경군을 그룹화하여 범례 토글로 묶음 처리.

### 2) 수집 주기/타임스탬프 정규화
- `FactoryData.Time` 또는 `Date.now()` 기준으로 타임스탬프 통일.
- Extruder/LS(0.2s)와 SPOT(3s) 샘플링 편차를 고려하여
  - 데이터 없는 구간은 `null`로 유지해 단절 표시
  - SPOT는 최신값 유지 대신, 샘플 시점만 기록(선 연결 끊김 유지)

### 3) 프론트 버퍼(슬라이딩 윈도우) 구현
- 시리즈별 링 버퍼(최근 N분, 예: 10~30분) 유지.
- 수집 주기 0.2s 기준, 30분이면 약 9,000 포인트/시리즈.
- 초과 시 구간 절삭(메모리 상한 고정).

### 4) DataFrame 변환 레이어 구성
- Grafana DataFrame 포맷으로 변환(필드: `time` + 각 시리즈 값).
- `null` 값은 그대로 유지하여 결측/지연을 시각적으로 노출.
- 동일 타임스탬프 기반으로 시리즈 병합.

### 5) Scene TimeSeries Panel 생성
- Grafana Scenes의 TimeSeries 패널(VizPanel 기반)로 구성.
- 기본 옵션:
  - legend = 리스트 + 토글
  - tooltip = 크로스헤어 + 다중 시리즈 표시
  - line width/point size는 공정 화면 가독성 기준으로 조정
  - span nulls = false (결측을 끊김으로 표시)

### 6) 다중 축/단위 적용
- 온도군 / 압력·속도군 / 환경군을 Y축 그룹으로 분리.
- 단위 변환(예: `At_Temp`, `At_Pre`는 /100 적용된 값 사용).
- 축별 색상/레이블 일관성 유지.

### 7) 임계값/상태 라인 연동
- 설정의 임계값 또는 기존 상태 로직을 TimeSeries 임계값 라인으로 연결.
- 예: 압력 180 이상, SPOT 580 이상 경고 등
- 임계값 라인은 색상으로 강하게 구분(운영자가 직관적으로 인지).

### 8) 사용자 인터랙션 구현
- Live/Pause 토글(실시간 흐름 정지/재개)
- 범례/그룹 토글(온도군/공정군/환경군)
- 드래그 줌/팬 + Reset
- 크로스헤어 툴팁(동기 표시)
- 최근 N분 프리셋(1/5/10/30/60)
- 선택 옵션: 임계값 라인 On/Off, 결측 표시 모드, 스무딩 토글

### 9) 업데이트 성능 최적화
- 렌더 주기 200~500ms로 묶어 UI 업데이트 스로틀.
- 대량 리렌더 방지: 데이터 버퍼 업데이트와 렌더 업데이트 분리.
- 시리즈 필터링/숨김 상태에서는 데이터 변환 스킵.

### 10) 레이아웃/가시성 저장
- 선택된 시리즈 그룹/창 크기(N분)/축 분리 상태를 레이아웃 저장에 포함.
- 레이아웃 복구 시 동일 상태로 재현되도록 설계.

### 11) 오류/지연 표시 정책 반영
- 데이터 갱신 지연 시 라벨 또는 상태 배지 표시(기존 규칙과 일치).
- `--` 표시 정책을 TimeSeries 범례/툴팁에도 반영.

## 4. 패널 구성 권장안(가독성 우선)
- 2~3개 패널로 분리 권장:
  - 공정 KPI 패널: `Press`, `Speed`, `EndPos`, `Count`, `Billet`
  - 온도 패널: `Spot`, `Temp_F`, `Temp_B`, `Billet_Temp`, `Mold1~6`, `At_Temp`
  - 환경 패널: `At_Pre`
- 단일 패널로 합치는 경우, 그룹 토글(온도군/공정군/환경군) 필수

## 5. 사용자 인터랙티브 기능 제안
- P0(필수): Live/Pause 토글, 범례/그룹 토글(온도군/공정군/환경군), 드래그 줌/팬 + Reset, 크로스헤어 툴팁, 최근 N분 프리셋(1/5/10/30/60)
- P1(중요): 시리즈별 마지막 갱신 시각 표시, 임계값 라인 On/Off, 결측 표시 모드(끊김/보간), 간단 스무딩(이동평균) 토글
- P2(선택): 알람 히스토리 오버레이, 스냅샷/CSV 내보내기, 과거 구간 비교 오버레이

## 6. 검증 체크리스트
- 5Hz 스트리밍 기준 CPU/메모리 급증 여부
- 결측 구간이 0으로 채워지지 않고 끊김으로 표시되는지
- 임계값 라인이 설정 값 변경 시 즉시 반영되는지
- 레이아웃 저장/복구 시 패널 상태(축, 가시성)가 유지되는지
