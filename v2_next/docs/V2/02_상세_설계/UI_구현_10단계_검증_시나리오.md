# UI 구현 10단계 검증 시나리오

## 1. 목표
- UI 구현 결과가 요구사항과 일치하는지 검증한다.

## 2. 검증 시나리오

### 2.1 레이아웃 검증
- 1920x1080에서 3컬럼 구조가 유지되는지 확인
- 카드 간격/정렬이 문서 기준과 일치하는지 확인
- 편집 모드 진입 시 그리드 배경(60열/20px/4px)이 표시되는지 확인

### 2.2 KPI 라벨 검증
- 속도 라벨: `>=8 매우빠름`, `6~<8 빠름`, `4~<6 보통`, `2~<4 저속`, `0~<2 매우저속`
- 압력 라벨: `>=180 높음`, `126~<180 보통`, `<126 낮음`
- 라벨 색상: 매우빠름=적색, 빠름=황색, 보통=청록, 저속=청색, 매우저속=회청색

### 2.3 SPOT 경고 검증
- 500°C 이상에서 정적 테두리 + 경고 아이콘 표시
- Arc Gauge가 위험 색상으로 전환되는지 확인

### 2.4 카메라 상태 검증
- `2 x refresh_interval` 초과 시 “지연됨” 라벨 표시
- `5 x refresh_interval` 초과 시 “연결 끊김” 표시
- 상태 오버레이에 텍스트 + 스피너가 표시되는지 확인

### 2.5 결측/지연/오류 검증
- 값 없음 시 `--` + 회색 처리
- 지연/오류 라벨 + 아이콘 표시
- 마지막 갱신 시각(`HH:MM:SS`)이 카드 하단에 노출되는지 확인

### 2.6 편집 모드 검증
- 편집 모드에서만 드래그/리사이즈 가능
- 저장 성공 시 비차단 알림 표시
- 저장 실패 시 오류 라벨/재시도 버튼 표시

## 3. 완료 기준
- 모든 시나리오가 재현 및 확인되면 UI 구현 완료로 판단한다.
