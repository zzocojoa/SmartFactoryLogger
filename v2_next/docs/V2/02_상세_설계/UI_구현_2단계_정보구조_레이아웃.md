# UI 구현 2단계 정보 구조·레이아웃 확정

## 1. 목표
- 1920x1080 기준의 대시보드 정보 구조와 카드 배치를 고정한다.
- 공정 흐름과 운영자 시선 동선(F-패턴)을 일치시킨다.

## 2. 전제 조건
- 화면 해상도: 1920x1080
- 기본 폰트: Pretendard
- 레이아웃은 3컬럼 기본 구조 유지

## 3. 레이아웃 구조(확정)

### 3.1 상단 바(Top Bar)
- 구성: 라인명, 현재 시각, Running/Degraded/Disconnected, 마지막 갱신 시각, 저장/편집 제어
- 높이: 28~32px
- 우측: 저장 상태, 편집 모드 토글, 설정 아이콘

### 3.2 좌측 컬럼(Zone A)
- **Process KPI** 카드
- 보조 KPI(카운트/종료 위치) 포함
- 시선 시작점 배치

### 3.3 중앙 컬럼(Zone B)
- **SPOT Temperature** 카드 (Arc Gauge + 정적 경고 강조)
- **Secondary Temps** 카드(컨테이너 전/후, 빌렛 온도/길이)
- **SPOT Camera** 카드(영상 + 오버레이, 연결 상태 텍스트+스피너)

### 3.4 우측 컬럼(Zone C)
- **Mold Zones** 카드(2x3 그리드)
- **Environment** 카드(At Temp/At Pre)
- **Operator Check** 카드(경고/조치 문구)

## 4. 배치 원칙
- 주요 시선 흐름: 좌상단 → 중앙 상단(SPOT) → 우측 상단(Mold) → 우측 하단(Operator Check) → 중앙 하단(Camera)
- 카드 간격: 12~16px, 컬럼 간격: 16~20px
- 카드 레이아웃은 데이터 변화와 무관하게 고정

## 5. 편집 모드 규칙
- 편집 모드에서만 이동/리사이즈 가능
- 편집 모드 시 그리드 스냅 가시화
- 저장 성공 메시지는 비차단형 표시

## 6. 결과
- 정보 구조와 카드 배치를 확정하였으며, 이후 단계는 이 구조를 기준으로 구현한다.
