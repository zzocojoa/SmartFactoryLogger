# 실시간 대시보드 UI 디자인 리포트 (알루미늄 압출 공정, 2025)

## 1. 개요 (Overview)

본 문서는 알루미늄 압출 공정의 실시간 모니터링을 위한 대시보드 UI/UX 설계
가이드라인을 정의한다. 현장의 데이터(속도, 압력, 온도 등)를 직관적으로
시각화하여 운영자가 공정 상태를 즉시 파악하고 이상 징후에 신속하게 대응할 수
있도록 하는 것을 목표로 한다. 특히, **Grafana Scenes** 엔진 도입에 따른 레이아웃
유연성과 인터랙션 가능성을 반영하였다.

## 2. 디자인 원칙 (Design Principles)

### 2.1 가시성 최우선 (Visibility First)

- **원거리 가독성**: 운영자가 모니터와 2~3m 떨어진 거리에서도 핵심 수치(압력,
  속도)를 식별할 수 있어야 한다.
- **데이터 계층화**: 모든 정보가 동등하지 않다. **Process KPI > SPOT 온도 > 상태
  알림** 순으로 시각적 위계를 설정한다.

### 2.2 산업 표준 준수 (Industrial Standard)

- **상태 색상**: ISO/IEC 표준에 준하는 상태 색상을 사용한다 (정상: Green, 경고:
  Amber, 위험: Red).
- **다크 모드 기본**: 제어실의 낮은 조도 환경과 장시간 모니터링 시의 눈 피로도를
  고려하여 **High Contrast Dark Mode**를 기본으로 채택한다.

### 2.3 인지 부하 최소화 (Minimal Cognitive Load)

- **정적 레이아웃**: 데이터 값의 변화(예: 10 → 100)로 인해 UI 레이아웃이
  흔들리거나 재정렬되는 것을 방지한다 (Tabular Nums 사용).
- **직관적 메타포**: 온도 상승은 적색 계열의 그라데이션, 압력/속도는 **숫자 + 엷은 가로 Bar**로 표현하여 직관적으로 파악하도록 한다.

### 2.4 데이터 안정성 (Data Stability)

- **자릿수/단위 고정**: 소수점 자릿수와 단위 위치를 고정하여 숫자 흔들림을 최소화.
- **결측 표시**: 값이 없으면 `--` + 회색 처리 + 마지막 갱신 시각 표기.
- **레이아웃 불변**: 데이터 길이가 변해도 카드 크기/정렬이 바뀌지 않도록 최소 폭/패딩 유지.

## 3. 정보 구조 및 레이아웃 (Information Architecture)

Grafana Scenes의 **Grid Layout** 시스템을 활용하여, 논리적 연관성에 따라 카드를
그룹화한다.

| 구역 (Zone)              | 구성 요소 (Components)                    | 디자인 의도 (Design Intent)                                                     |
| :----------------------- | :---------------------------------------- | :------------------------------------------------------------------------------ |
| **Top Bar**              | 타이틀, 시스템 상태, 마지막 갱신 시각, 저장/편집 제어 | 전체 시스템의 헬스 체크 및 글로벌 컨트롤 제공                                   |
| **Zone A (좌상단)**      | **Process KPI** (속도, 압력, 카운트)      | 공정의 "맥박"에 해당하는 핵심 지표를 F-패턴의 시작점에 배치                     |
| **Zone B (우상단/중단)** | **Temperature Group** (SPOT, 2차 온도)    | 열적 데이터(Thermal Data)를 한곳에 모아 온도 흐름 상관관계 파악 용이하도록 배치 |
| **Zone C (우하단)**      | **Mold Zones** (금형 온도 1~6)            | 다수의 금형 데이터를 2x3 그리드로 정형화하여 패턴 인식 유도                     |
| **Zone D (좌하단)**      | **Visual Monitoring** (SPOT 카메라, 알림) | 정량적 수치 보완을 위한 시각 정보 및 텍스트 기반 조치 사항 제공                 |

## 4. 상세 컴포넌트 디자인 가이드

### 4.1 타이포그래피 (Typography)

- **Font Family**: 가독성이 뛰어난 산세리프(San-serif) 서체 사용 (예: Pretendard, IBM Plex Sans KR).
- **Tabular Figures (등폭 숫자)**: 실시간으로 빠르게 변하는 숫자의 흔들림을 막기
  위해 고정폭 숫자 속성(`font-variant-numeric: tabular-nums`) 적용 필수.
- **크기 계층**:
  - **Hero Data**: 48px+ (속도, 대표 온도)
  - **Primary Data**: 24px~32px (보조 온도, 압력)
  - **Label**: 14px~16px (단위, 항목명 - 약한 대비 색상 적용)

### 4.2 색상 팔레트 (Color Palette)

압출 공정의 특성(고열)을 반영한 팔레트를 정의한다.

- **Background**: `#111217` (Deep Dark Grey) - 눈의 피로 감소
- **Card Background**: `#181b1f` (Card Grey) - 배경과 구분되는 레이어
- **Primary Accent**: `#56A6DC` (Information Blue)
- **Heat Gradient**: `#FF9966` → `#FF4444` (온도 상승 시각화용)
- **Status Colors**:
  - **Normal**: `#73BF69` (Grafana Green)
  - **Warning**: `#FF9F30` (Grafana Orange)
  - **Critical**: `#F2495C` (Grafana Red)
- **High Performance HMI 원칙**: 정상 상태는 저채도, 색상은 이상 상태에 집중

### 4.3 위젯별 디자인 전술

1. **Process KPI**:
   - 숫자 크기를 극대화하고 단위를 작게 표기하여 가독성 확보.
   - 전일/목표 대비 증감 표시(▲/▼)를 통해 추세 정보 제공.
   - **얇은 가로 Bar**로 변화 정도를 보조 표현(저자극, 고가독성).
2. **SPOT Temperature**:
   - 단순 수치뿐만 아니라, 임계치(Threshold) 도달 정도를 보여주는 **Arc
     Gauge**를 배경 은유로 활용.
   - 500°C 이상 과열 시 **정적 테두리 + 경고 아이콘**으로 강조(점멸 지양).
3. **Camera View**:
   - 카메라 영상 위에 SVG 오버레이로 조준점(Crosshair)을 명확히 표시.
   - 영상/연결 지연 시 “연결 중” 텍스트 + 스피너를 중앙 고정 표시.
4. **Mold Grid**:
   - 6개의 몰드 온도를 "작은 카드"의 반복 패턴으로 배치.
   - 특정 몰드 이상 발생 시 해당 셀만 Highlight 처리하여 "이상 징후"를
     팝업(Pop-up) 효과로 전달.

## 5. 인터랙션 및 UX (Grafana Scenes)

### 5.1 편집 모드 (Edit Mode)

- **Drag & Drop**: 운영자가 자신에게 편한 위치로 카드를 이동할 수 있는 자유도
  제공 (단, 실수 방지를 위해 드래그 핸들 영역 명시적 제공).
- **Resizing**: 중요도에 따라 카드의 크기를 조절할 수 있는 유연성 제공.
- **스냅/정렬**: 그리드 스냅과 충돌 방지로 겹침/등싱을 최소화하고, 편집 모드에서만 시각화.
- **Persistence**: 변경된 레이아웃은 로컬 스토리지에 즉시 저장되어, 재접속
  시에도 개인화된 화면 유지.

### 5.2 상태 피드백 (State Feedback)

- **Hover Effects**: 카드에 마우스 오버 시 미세한 밝기 변화(Lift effect)를 주어
  "클릭/조작 가능함"을 암시.
- **연결/로딩 표시**: "연결 중" 텍스트 + 스피너를 표시하여 상태 인지를 명확하게 함.
- **비차단 알림**: 저장/경고 메시지는 Toast/배지 형태로 표시하여 작업 동선을 방해하지 않도록 한다.

## 6. 데이터/알람 표준 권고

- **ISA-101 / High Performance HMI**: 정상 상태는 저채도 중심, 색상은 이상 상태 표시에 집중.
- **ISA-18.2 / EEMUA 191**: 경고 등급화, 중복 경고 억제, 최소 표시 시간(3~5초) 유지 권고.
- **자릿수 정책**: 속도/압력/온도는 1자리, 카운트는 정수 고정.
- **단위 일관성**: 단위 표기 위치를 고정하고 다중 화면에서도 동일 규칙 유지.

### 6.1 KPI/카메라 임계값 정책 (v2 운영 기준)

- **공통 원칙**: 0.2초 주기 기준, 단일 샘플에 반응하지 않고 2~5초 지속 시 상태 전환.
- **Speed/Press**
  - Warning: `min_speed`/`max_speed`, `min_press`/`max_press` 이탈이 2~3초 지속.
  - Danger: `speed == 0` + `press >= press_running_threshold`가 2~3초 지속(정지/막힘).
  - 기준값이 없으면 최근 5분 중앙값 대비 ±30%(Warning) / ±50%(Danger) 적용.
- **Count**
  - Warning: `speed > min_speed` && `press > press_running_threshold`인데도 10초 이상 증가 없음.
  - Danger: 카운트 역행/비정상 리셋(정상 사이클 리셋 구간 제외).
- **End Position**
  - Warning: 허용 범위 이탈이 2~3초 지속.
  - Danger: 급격한 역방향 점프(허용 오차 초과).
- **SPOT Camera**
  - Warning: 새 프레임 미도착이 `2 x refresh_interval` 초과 또는 연속 오류 3회 이상.
  - Danger: 새 프레임 미도착이 `5 x refresh_interval` 초과 또는 연속 오류 10회 이상.

## 7. 결론 및 제언

본 디자인 리포트는 알루미늄 압출 공정의 특수성(고열, 고압, 빠른 사이클)과 운영
환경(공장 제어실)을 깊이 있게 고려하였다. 단순한 "예쁜 화면"이 아닌, **"오판단을
줄이고 반응 속도를 높이는"** 기능적 미학(Functional Aesthetic)을 추구해야 한다.
Grafana Scenes의 모던한 기능을 십분 활용하여, 정적이면서도 역동적인(데이터
반응형) 대시보드를 구축할 것을 권장한다.
