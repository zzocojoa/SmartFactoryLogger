# UI 구현 4단계 컴포넌트 스펙 확정

## 1. 목표
- 컴포넌트별 데이터/레이아웃/상태 규칙을 최종 확정한다.

## 2. 확정 대상
- 카드 구성 요소, 데이터 바인딩, 상태 라벨 규칙
- KPI 속도/압력 라벨 체계
- SPOT 경고 강조 방식
- 카메라 연결/지연 표시 방식

## 3. 확정 내용

### 3.1 KPI 컴포넌트
- **표시 항목**: Speed, Press, Count, EndPos
- **라벨 체계(속도)**: `>=8 매우빠름`, `6~<8 빠름`, `4~<6 보통`, `2~<4 저속`, `0~<2 매우저속`
- **라벨 체계(압력)**: `>=180 높음`, `126~<180 보통`, `<126 낮음`
- **색상**: 매우빠름=적색, 빠름=황색, 보통=청록, 저속=청색, 매우저속=회청색
- **보조 표현**: 얇은 가로 Bar
- **경고 매핑**: Normal/Warning/Danger 대신 라벨 기반 표현 유지

### 3.2 SPOT Temperature
- **표시 항목**: Spot
- **시각화**: Arc Gauge + 대형 수치
- **위험 기준**: 500°C 이상
- **강조 방식**: 정적 테두리 + 경고 아이콘 (점멸 금지)

### 3.3 Secondary Temps
- **표시 항목**: Temp_F, Temp_B, Billet_Temp, Billet_Length
- **레이아웃**: 2x2 그리드
- **포맷**: 온도 1자리, 길이 0~1자리

### 3.4 Mold Zones
- **표시 항목**: Mold1~Mold6
- **레이아웃**: 2x3
- **강조 방식**: 이상 셀만 강조
- **기준**: v1 기준값 유지(`>= 100` 이상 시 강조)

### 3.5 Environment
- **표시 항목**: At_Temp, At_Pre
- **라벨 체계(온도)**: `<10 추움`, `10~<28 쾌적`, `>=28 더움`
- **라벨 체계(습도)**: `<30 건조`, `30~<60 쾌적`, `>=60 다습`
- **표시 방식**: 소형 2열 카드 + 상태 배지

### 3.6 SPOT Camera
- **표시 항목**: Spot 이미지, 크로스헤어
- **상태 표시**: “연결 중” 텍스트 + 스피너 중앙 고정
- **지연 기준**: `2 x refresh_interval` Warning, `5 x refresh_interval` Danger

### 3.7 Operator Check
- **표시 항목**: 경고 메시지
- **강조 방식**: 테두리/배경 강조 + 아이콘

## 4. 확정 결과
- 위 스펙은 코드 구현 및 스타일 적용의 기준으로 사용한다.
