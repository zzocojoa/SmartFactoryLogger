# 카드별 UI 상세 설계 (압출 공정 실시간 대시보드, 2025)

## 1. 적용 대상
- 카드형 대시보드(공정 KPI, SPOT, 온도군, 몰드, 환경, 카메라, 알림)
- 운영 모드(가동/정지), 장애 모드(연결 끊김/값 결측) 표시 포함

## 2. 공통 UI 규격
- **그리드 기준**: 24 컬럼 기준, 카드 간 간격 12~16px 권장.
- **최소 크기**: 숫자가 깨지지 않는 최소 카드 크기 지정(예: KPI/온도 카드 최소 4x3, 카메라/알림 카드 최소 6x5).
- **타이포 스케일**: 주요 수치 `clamp(24px, 4vw, 40px)`, 보조 텍스트 12~14px.
- **숫자 안정성**: `tabular-nums` 적용, 소수점 자릿수 고정(예: 1자리).
- **단위 표기**: 수치 오른쪽에 고정(예: `430.5 °C`), 위치/크기 통일.
- **상태 표시**: 색상 + 텍스트 + 아이콘 동시 사용(색상만 사용 금지).
- **결측 처리**: 값 없을 때 `--` + 회색, 마지막 갱신 시간 표시.
- **편집 모드 분리**: 이동/리사이즈는 편집 모드에서만 활성화.
- **임계값 지속성**: 단일 샘플이 아니라 2~5초 지속 시 상태 전환.

## 3. 카드별 상세 설계

### 3.1 공정 KPI 카드 (속도/압력/카운트/종료 위치)
- **권장 크기**: 기본 6x8, 최소 5x6.
- **레이아웃**: 상단 제목, 중앙 2열 수치(속도/압력), 하단 보조(카운트/종료 위치).
- **시각 규칙**: 수치는 크게, **얇은 가로 Bar**로 변화 정도를 보조 표현.
- **상태 규칙**: 속도 0 또는 압력 0 지속 시 “정지” 배지.
- **수치 포맷**: 속도 1자리, 압력 1자리, 카운트는 정수.
- **임계값 기준**: `min/max` 이탈 2~3초 지속 시 Warning, `speed==0` + `press>=threshold` 2~3초 지속 시 Danger.
- **라벨 규칙**: `>=8 매우빠름`, `6~<8 빠름`, `4~<6 보통`, `2~<4 저속`, `0~<2 매우저속`.
- **압력 라벨**: `>=180 높음`, `126~<180 보통`, `<126 낮음`.
- **라벨 색상**: 매우빠름=적색, 빠름=황색, 보통=청록, 저속=청색, 매우저속=회청색(경고 색상과 분리).

### 3.2 SPOT Temperature 카드
- **권장 크기**: 기본 10x4, 최소 8x4.
- **레이아웃**: 중앙 대형 수치 + 하단 상태 라벨(정상/주의/위험).
- **임계치**: **Arc Gauge**로 임계치 도달 정도 표시(상태 라벨 병행).
- **변화 강조**: 이전 값 대비 상승/하락 아이콘(▲/▼)을 숫자 옆에 표시.
- **경고 강조**: 500°C 이상 시 **정적 테두리 + 경고 아이콘** 사용(점멸 지양).

### 3.3 Secondary Temps 카드 (컨테이너 전/후, 빌렛 온도/길이)
- **권장 크기**: 기본 10x4, 최소 8x4.
- **레이아웃**: 2x2 미니 카드 그리드.
- **포맷**: 온도는 1자리, 길이는 정수 또는 1자리.
- **우선순위**: 온도 항목을 상단 라인에 배치.

### 3.4 Mold Zones 카드 (Mold 1~6)
- **권장 크기**: 기본 8x8, 최소 7x6.
- **레이아웃**: 3x2 그리드 + 몰드 번호 라벨 고정.
- **이상값 강조**: 해당 몰드만 테두리와 배경 강조, 나머지는 유지.
- **집중성**: 평균/최댓값 요약 배지를 우측 상단에 소형으로 배치.

### 3.5 환경 카드 (At Temp / At Pre)
- **권장 크기**: 기본 4x2~4x3, 최소 4x2.
- **레이아웃**: 수치 + 상태 라벨(쾌적/건조/주의).
- **시각 톤**: 공정 수치보다 시각적 강도를 낮게 유지.

### 3.6 SPOT Camera 카드
- **권장 크기**: 기본 12x10, 최소 8x6.
- **레이아웃**: 상단 제목, 중앙 영상, 하단 컨트롤(포커스).
- **오버레이**: 크로스헤어/포커스 표시, 로딩/연결 상태는 중앙 고정(텍스트 + 스피너).
- **비율 유지**: 영상 비율 고정(16:9 또는 카메라 실제 비율).
- **지연/끊김 기준**: `2 x refresh_interval` 초과는 Warning, `5 x refresh_interval` 초과는 Danger.

### 3.7 알림/Operator Check 카드
- **권장 크기**: 기본 12x10, 최소 8x6.
- **레이아웃**: 상단 경고 아이콘 + 제목, 본문은 행동 지시 문장.
- **강조 규칙**: 경고 발생 시 테두리/배경 강조, 비활성 시 톤 다운.
- **지속성**: 경고는 최소 표시 시간 확보(예: 3~5초 유지).

## 4. 반응형/리사이즈 정책
- **최소 크기 이하**: 보조 텍스트 숨김, 핵심 수치만 표시.
- **확대 시**: 수치 텍스트만 확대, 카드 자체는 고정 크기 유지.
- **글자 잘림 방지**: 단위는 줄바꿈 금지, 숫자 영역은 넉넉한 패딩 확보.

## 5. 상태/에러 시나리오
- **연결 끊김**: 카드 전체 톤 다운 + “연결 끊김” 라벨.
- **데이터 지연**: “지연됨(마지막 갱신 HH:MM:SS)” 표시.
- **센서 오류**: 해당 카드에만 오류 배지 표시, 나머지 카드 유지.

## 6. 구현 체크리스트 (UI 관점)
- [ ] 모든 카드에 단위 표기/자릿수 규칙 적용
- [ ] 최소 크기에서 텍스트가 잘리지 않는지 확인
- [ ] 드래그/리사이즈는 편집 모드에서만 활성화
- [ ] 경고/알림은 색상+텍스트+아이콘 동시 사용

## 7. 결론
카드별 규격을 고정하면 운영자 관점에서 “즉시 판단”이 가능해지며, 리사이즈나 데이터 갱신으로 인한 흔들림을 최소화할 수 있다. 특히 SPOT/몰드/카메라 카드의 시각적 계층을 명확히 하면 현장 대응 속도가 개선된다.
