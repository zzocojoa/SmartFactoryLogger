# 설정 UI 적용 순서 맵 (App.tsx 기준, 2025)

## 1. 현재 모달 구조 요약(App.tsx)
- `settings-backdrop` → `settings-modal`
- 헤더: `settings-header`(제목/닫기)
- 상태: `settings-status`, `settings-error`, `settings-info`
- 본문: `settings-form`
  - `통신 설정`(Extruder/LS PLC/SPOT IP/Port/Refresh)
  - `저장 설정`(log/snapshot path, autoSave)
  - `로그 회전`(rotation mode, cycle idle/threshold)
  - `보안`(비밀번호)
- 하단: `settings-footer`(안내 문구 + 저장/닫기)

## 2. 적용 순서(의존성 기준)

### 1단계: 상단 상태 바 확장
- 기존 `settings-info`를 상단 고정 바 형태로 확장.
- 항목: 설정 파일 경로, 변경 카운트, 재시작 필요 배지.
- 이유: 이후 변경 추적/저장 여부 표시의 기준점.

### 2단계: 변경 사항 추적(Dirty State)
- 입력 필드 변경 시 변경 카운트 업데이트.
- 변경된 항목 하이라이트.
- 저장 전 확인 다이얼로그(변경 요약 + 영향 범위) 추가.

### 3단계: 섹션 네비게이션 도입
- 좌측 섹션 목록(요약/통신/카메라/저장/로그/보안/고급).
- 변경 발생 섹션 배지 표시.
- 좁은 화면에서는 드롭다운으로 축소.

### 4단계: 요약 섹션 카드 추가
- 통신/저장/SPOT 요약 카드 3종 추가.
- 최근 저장 시각 + 백업(.bak) 상태 표시.
- 사용 목적: 빠른 현장 확인.

### 5단계: 통신/장비 섹션 고도화
- IP/Port 유효성 검사.
- 장비별 연결 테스트 버튼 + 결과 배지 + 최근 시각.
- 기존 통신 설정 영역에 바로 적용.

### 6단계: SPOT 카메라 섹션 분리
- 기존 통신 섹션에서 SPOT 관련 항목 분리.
- 미리보기 URL 읽기 전용 표시.
- 수신 지연/오류 배지 표시.

### 7단계: 저장/로깅 섹션 검증
- 로그/스냅샷 경로 검증(존재/쓰기 권한).
- NAS(Z:) 경로 감지 배지 표시.
- 저장 불가 시 저장 버튼 비활성화.

### 8단계: 로그 회전/알림 섹션 확장
- 로그 회전 설명 마이크로카피 추가.
- (옵션) 알림/임계값 섹션 추가.

### 9단계: 보안/권한 섹션 강화
- 비밀번호 상태 배지(설정됨/미설정).
- 읽기 전용 감지 시 안내 메시지 및 저장 비활성화.

### 10단계: 하단 액션 바 확장
- 기본값 복원, 백업 복원 버튼 추가.
- 저장 성공/실패 토스트 + 재시작 필요 배지 유지.

## 3. 최소 변경으로 시작할 경우(Quick Win)
- 1단계(상단 상태 바 확장)
- 2단계(변경 추적)
- 5단계(연결 테스트)
- 7단계(경로 검증)

## 4. 구현 시 주의사항
- 입력 검증 실패 시 저장 비활성화는 필수.
- 재시작 필요 항목은 저장 이후에도 배지를 유지.
- NAS 경로 오류는 현장 문제와 직결되므로 우선순위 상향.
