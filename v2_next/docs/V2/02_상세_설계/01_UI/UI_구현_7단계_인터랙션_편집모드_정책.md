# UI 구현 7단계 인터랙션·편집 모드 정책

## 1. 목적
- 운영 모드/편집 모드 구분으로 현장 오조작을 방지한다.
- 레이아웃 저장/복구/삭제 UX를 일관되게 제공한다.

## 2. 모드 정의
- **운영 모드**: 기본 상태. 카드 이동/리사이즈 비활성.
- **편집 모드**: 카드 이동/리사이즈 활성. 배경 그리드 표시.

## 3. 편집 모드 규칙
- 드래그 핸들은 카드 헤더 영역만 노출.
- 리사이즈 핸들은 우하단에만 노출.
- 편집 모드에서만 레이아웃 저장/복구/삭제 메뉴 표시.
- 편집 모드 토글 후 메뉴는 유지되어 바로 저장/복구를 선택 가능.

## 4. 저장/복원/삭제 정책
- 저장: 이름 입력 후 서버 저장, 최대 3개 슬롯.
- 복구: 목록에서 선택 → 활성 레이아웃 변경.
- 삭제: 목록에서 삭제 → 활성 슬롯 삭제 시 첫 슬롯 또는 null로 재설정.
- 비상 복구: `/api/layout/restore` 호출(`layout.backup.json` 기반).
- 로컬 스토리지는 1회 마이그레이션 용도로만 사용.

## 5. UX 피드백
- 저장/복구/삭제 성공: 비차단 메시지.
- 실패: 오류 라벨 + 재시도 버튼.
- 편집 모드 해제 시 드래그/리사이즈 즉시 비활성.

## 6. 결과
- 편집 모드 정책을 확정하고 구현에 반영한다.
