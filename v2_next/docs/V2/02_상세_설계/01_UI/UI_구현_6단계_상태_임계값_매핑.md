# UI 구현 6단계 상태·임계값 매핑

## 1. 목표
- 카드별 상태 전환 조건과 표시 규칙을 일관되게 매핑한다.

## 2. 공통 원칙
- 단일 샘플 기준이 아니라 **2~5초 지속** 기준으로 상태 전환.
- 라벨(속도/압력)은 **Normal/Warning/Danger로 치환하지 않음**.
- 경고 시각화는 **정적 강조(테두리/아이콘)** 중심, 점멸 금지.

## 3. 카드별 매핑

### 3.1 KPI
- **Speed 라벨**
  - `>= 8` 매우빠름
  - `6~<8` 빠름
  - `4~<6` 보통
  - `2~<4` 저속
  - `0~<2` 매우저속
- **Press 라벨**
  - `>= 180` 높음
  - `126~<180` 보통
  - `<126` 낮음
- **상태 전환(카드 강조)**
  - 정지/막힘 의심: `speed == 0` + `press >= 20` 2초 지속
  - 정지/막힘 확정: 위 조건 5초 지속

### 3.2 SPOT
- **Danger**: `spot >= 500°C` 2초 지속
- **표현**: 정적 굵은 테두리 + 경고 아이콘 + Arc Gauge 위험색

### 3.3 Container Temps
- **Warning**: `Temp_F/Temp_B > 350°C` 2~3초 지속
- **Danger**: `Temp_F/Temp_B > 450°C` 2~3초 지속

### 3.4 Billet Temp
- **Warning**: `Billet_Temp > 440°C` 2~3초 지속
- **Danger**: `Billet_Temp > 480°C` 2~3초 지속

### 3.5 Mold Zones
- **강조**: `>= 100` 이상 시 해당 셀만 강조

### 3.6 Environment
- **온도 라벨**: `<10 추움`, `10~<28 쾌적`, `>=28 더움`
- **습도 라벨**: `<30 건조`, `30~<60 쾌적`, `>=60 다습`

### 3.7 SPOT Camera
- **Warning**: 새 프레임 미도착 `2 x refresh_interval` 초과 또는 연속 오류 3회
- **Danger**: 새 프레임 미도착 `5 x refresh_interval` 초과 또는 연속 오류 10회
- **표현**: 텍스트 + 스피너 오버레이

### 3.8 Operator Check
- **Warning/Danger**: 상위 알림 상태에 연동
- **표현**: 테두리/배경 강조 + 아이콘

## 4. 확정 결과
- 위 매핑을 기준으로 UI 상태 표시 로직을 구현한다.
