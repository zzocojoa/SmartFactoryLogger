# 프론트엔드 UI 스타일 가이드 (압출 공정 대시보드, 2025)

## 1. 목적
- 카드형 대시보드의 색상/타이포/상태 표현을 통일하여 운영자 가독성과 신뢰성을 높인다.
- 디자인 토큰(변수) 기반으로 유지보수성을 확보한다.

## 2. 기본 디자인 토큰 (CSS 변수 예시)
```css
:root {
  --bg-main: #1b1f24;
  --bg-card: #242a30;
  --bg-card-muted: #1f2328;
  --text-primary: #e6edf3;
  --text-secondary: #b0bac4;
  --text-muted: #7b8794;

  --state-ok: #25c49a;
  --state-warn: #f2c94c;
  --state-danger: #f66b6b;
  --state-offline: #7a7a7a;
  --state-cool: #56a6dc;

  --card-radius: 12px;
  --card-padding: 14px;
  --card-gap: 12px;

  --font-number: "IBM Plex Sans KR", "Pretendard", "Segoe UI", sans-serif;
  --font-ui: "Pretendard", "Segoe UI", sans-serif;
}
```

## 3. 타이포 규칙
- **수치**: `font-variant-numeric: tabular-nums;`
- **주요 수치**: `clamp(24px, 4vw, 40px)`
- **보조 텍스트**: `12~14px`
- **단위**: 숫자 우측 고정, 줄바꿈 금지.

## 4. 카드 공통 스타일
```css
.card {
  background: var(--bg-card);
  color: var(--text-primary);
  border-radius: var(--card-radius);
  padding: var(--card-padding);
  box-shadow: 0 6px 18px rgba(0, 0, 0, 0.35);
}

.card--muted {
  background: var(--bg-card-muted);
  color: var(--text-secondary);
}
```

## 5. 상태 표현 (공통 클래스)

## 6. KPI 라벨 색상 규칙
```css
.label-speed-very-fast { color: var(--state-danger); }
.label-speed-fast { color: var(--state-warn); }
.label-speed-normal { color: var(--state-ok); }
.label-speed-slow { color: var(--state-cool); }

.label-press-high { color: var(--state-warn); }
.label-press-normal { color: var(--state-ok); }
.label-press-low { color: var(--state-cool); }
```

## 7. 카드별 보조 규칙

### 6.1 KPI 카드
- 큰 숫자 강조, 보조 수치 톤 다운.
- **얇은 가로 Bar**로 변화 정도를 보조 표현(높이 4~6px 권장).
- 상태(정지/가동) 배지 추가.

### 6.2 SPOT 카드
- 중앙 수치 + **Arc Gauge** 배경 + 하단 상태 라벨.
- 위험 시 **정적 테두리 + 경고 아이콘**으로 강조(점멸 지양).

### 6.3 온도군 카드
- 2x2 서브 그리드, 항목 간 대비 확보.
- 온도 항목은 더 밝은 색.

### 6.4 몰드 카드
- 동일 크기 그리드, 이상값만 강조.
- 경고 몰드는 배경 톤 + 테두리 강조.

### 6.5 카메라 카드
- 영상 영역은 카드 내부에서 비율 고정.
- 오류/연결 오버레이는 중앙 고정(텍스트 + 스피너).

### 6.6 알림 카드
- 아이콘 + 제목 + 행동 지시문.
- 경고 시 테두리/배경 강조, 비활성 시 톤 다운.

## 8. 편집 모드 (레이아웃 편집 UX)
- 편집 모드일 때만 드래그/리사이즈 핸들 표시.
- 편집 모드 배경 그리드 표시(스냅 인지 강화).

## 9. 접근성/운영 고려
- 색상만으로 상태 구분 금지(아이콘+텍스트 병행).
- 주요 경고는 최소 3초 이상 유지.
- 값 결측 시 “--”와 함께 회색 처리.
- **상태 클래스 적용 기준**: KPI/카메라 임계값은 2~5초 지속 조건을 기본으로 하며, Warning/Danger 전환은 토큰 색상에만 의존하지 않는다.

## 10. 결론
상태/색상/타이포 규칙을 고정하면 UI 일관성과 현장 대응 속도가 동시에 개선된다. 스타일 가이드는 컴포넌트 구현의 기준선으로 사용한다.
