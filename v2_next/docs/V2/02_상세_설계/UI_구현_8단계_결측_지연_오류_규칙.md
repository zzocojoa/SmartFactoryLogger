# UI 구현 8단계 결측·지연·오류 표시 규칙

## 1. 목표
- 데이터 결측/지연/오류 상태를 운영자가 즉시 이해하도록 표준화한다.

## 2. 기본 원칙
- 결측 시 `--` 표기 + 회색 톤 + 마지막 갱신 시각 표기.
- 지연/오류는 색상 + 아이콘 + 텍스트를 동시에 사용.
- 화면 흔들림 방지(텍스트 길이/단위 고정).

## 3. 상태별 규칙

### 3.1 결측(Missing)
- 조건: 값 없음 또는 파싱 실패
- 표시: `--` + 회색 텍스트 + “마지막 갱신 HH:MM:SS”
- 카드 배경/크기 변화 없음
- 구현 반영: 카드 하단 `missing-note`로 마지막 갱신 시각 노출

### 3.2 지연(Delayed)
- 조건: 갱신 주기 2배 초과
- 표시: “지연됨” 라벨 + 경고 아이콘 + 경고 색상
- 데이터는 마지막 유효 값 유지

### 3.3 오류(Error)
- 조건: 통신 오류, 파싱 오류, 연속 실패
- 표시: “연결 끊김” 라벨 + 아이콘 + 위험 색상
- 해당 카드만 강조, 전체 화면 영향 없음

## 4. 카메라 규칙
- Warning: `2 x refresh_interval` 초과 시 “지연됨”
- Danger: `5 x refresh_interval` 초과 시 “연결 끊김”
- 표시: 텍스트 + 스피너 오버레이
- 구현 반영: Warning/Danger/Loading/Error 오버레이 색상 분리

## 5. 완료 기준
- 결측/지연/오류 상태가 UI 전반에서 동일 규칙으로 표시됨
