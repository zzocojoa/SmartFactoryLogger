# UI 구현 3단계 카드별 UI 규격 정의

## 1. 목표
- 카드별 화면 구성 요소와 규격을 확정하여 구현 기준을 고정한다.

## 2. 전제
- 해상도: 1920x1080
- 폰트: Pretendard
- 편집 모드에서만 이동/리사이즈 허용

## 3. 카드별 규격

### 3.1 Process KPI
- 크기: 기본 6x8, 최소 5x6
- 구성: 속도/압력(대형 수치) + 카운트/종료 위치(보조)
- 보조 표현: **얇은 가로 Bar**
- 속도 라벨: `>=8 매우빠름`, `6~<8 빠름`, `4~<6 보통`, `2~<4 저속`, `0~<2 매우저속`
- 압력 라벨: `>=180 높음`, `126~<180 보통`, `<126 낮음`
- 상태 전환: 라벨 기반 표현(Warning/Danger 매핑 없이 운영)
- 라벨 색상(직관형):
  - 속도: 매우빠름=적색, 빠름=황색, 보통=청록, 저속=청색, 매우저속=회청색
  - 압력: 높음=황색, 보통=청록, 낮음=청색

### 3.2 SPOT Temperature
- 크기: 기본 10x4, 최소 8x4
- 구성: 대형 수치 + **Arc Gauge** + 상태 라벨
- 경고 강조: 500°C 이상 시 **정적 테두리 + 경고 아이콘**

### 3.3 Secondary Temps
- 크기: 기본 10x4, 최소 8x4
- 구성: 2x2 서브 그리드(컨테이너 전/후, 빌렛 온도/길이)
- 포맷: 온도 1자리, 길이 0~1자리

### 3.4 Mold Zones
- 크기: 기본 8x8, 최소 7x6
- 구성: 2x3 그리드, 이상 셀만 강조
- 규칙: 전체 카드 색상 변화 최소화

### 3.5 Environment (At Temp/At Pre)
- 크기: 기본 4x3, 최소 4x2
- 구성: 소형 카드 2개 또는 2열 배치
- 라벨: 쾌적/보통/낮음

### 3.6 SPOT Camera
- 크기: 기본 12x10, 최소 8x6
- 구성: 영상 + 크로스헤어 + 상태 오버레이
- 상태: “연결 중” 텍스트 + 스피너 중앙 고정

### 3.7 Operator Check
- 크기: 기본 12x10, 최소 8x6
- 구성: 아이콘 + 제목 + 행동 지시 문장
- 강조: 경고 시 테두리/배경 강조, 비활성 시 톤 다운

## 4. 완료 기준
- 카드별 크기/구성이 문서 기준과 일치
- SPOT 경고 강조가 정적 테두리+아이콘 방식으로 적용
- KPI 라벨은 Normal/Warning/Danger가 아닌 사용자 정의 라벨로 표시
