# UI 구현 1단계 요구사항·우선순위 정리

## 1. 목적
- UI 구현 착수 전 요구사항을 고정하고 우선순위를 합의하기 위한 기준 문서

## 2. 우선순위 결정 기준
- **P0**: 안전/오판 방지, 운영 중단 위험과 직결
- **P1**: 운영 효율·인지 속도 개선
- **P2**: 품질 향상·편의성 강화

## 3. 요구사항 목록 (우선순위)

### P0 (필수)
- 원거리 가독성 확보(2~3m 기준)
- 자릿수·단위 고정 규칙
- 상태 표시 3중 규칙(색상+아이콘+텍스트)
- 점멸 금지 및 정적 강조 원칙
- 결측/지연 표시 규칙(`--` + 타임스탬프)
- 카드 레이아웃 흔들림 방지
- SPOT 500°C 이상 위험 강조(정적 테두리+아이콘)
- 카메라 연결/지연 상태 표시(텍스트+스피너)
- 편집 모드 분리(이동/리사이즈 제한)
- 레이아웃 저장/복원 안정성

### P1 (중요)
- KPI 수치 + 얇은 가로 Bar 보조 표현
- SPOT Arc Gauge 시각화
- 몰드 셀 단위 강조 규칙
- 알림/오퍼레이터 체크 행동 지시 문구 규칙
- 상태 바/알림 센터 연동 체계

### P2 (선택)
- 스파크라인·추세 시각화
- 사용자 개인화 프리셋
- 테마 변형(저채도/고채도 옵션)

## 4. 결정 사항
- 현장 모니터 해상도: 1920x1080
- 기본 폰트: Pretendard
- KPI 임계값(운영 기준 반영):
  - 속도: `>= 8` 매우빠름, `6~<8` 빠름, `4~<6` 보통, `2~<4` 저속, `0~<2` 매우저속
  - 압력: `>= 180` 높음, `126~<180` 보통, `<126` 낮음

## 5. 추가 확인 필요
- 속도/압력 구간을 Normal/Warning/Danger로 매핑하지 않고 구간 라벨 그대로 사용
