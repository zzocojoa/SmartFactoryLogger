# UI 구현 7단계 인터랙션·편집 모드 정책

## 1. 목표
- 운영 모드와 편집 모드를 명확히 분리하여 오조작을 방지한다.

## 2. 모드 정의
- **운영 모드**: 기본 상태. 카드 이동/리사이즈 비활성.
- **편집 모드**: 카드 이동/리사이즈 활성. 그리드 스냅 시각화.

## 3. 편집 모드 규칙
- 드래그 핸들은 상단 헤더 영역에만 노출.
- 리사이즈 핸들은 우측 하단에만 노출.
- 편집 모드에서만 그리드 배경 표시.
- 저장 버튼은 편집 모드에서만 활성화.

## 4. 저장/복원 정책
- 저장 성공 시 비차단형 알림(Toast/배지) 노출.
- 저장 실패 시 오류 라벨 + 재시도 버튼 표시.
- 로컬 스토리지 키는 고정(`grafana_scene_layout_v1`).

## 5. 사용자 피드백
- 카드 hover는 운영 모드에서만 미세 강조.
- 위험 경고는 편집 모드에서도 유지(색상/아이콘 고정).

## 6. 결과
- 편집 모드 정책을 확정하고, 이후 구현 단계에서 반영한다.
