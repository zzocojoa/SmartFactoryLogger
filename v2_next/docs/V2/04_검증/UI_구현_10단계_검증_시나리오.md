# UI 구현 10단계 검증 시나리오

## 1. 목표
- UI 구현 결과가 요구사항에 맞게 동작하는지 검증한다.

## 2. 검증 시나리오

### 2.1 레이아웃 검증
- 1920x1080에서 3컬럼 레이아웃이 유지되는지 확인
- 카드 간 간격/정렬이 문서 기준과 일치하는지 확인
- 편집 모드에서 배경 그리드(60x20, 4px 보조) 표시 확인

### 2.2 KPI 라벨 검증
- 속도 라벨: `>=8 매우빠름`, `6~<8 빠름`, `4~<6 보통`, `2~<4 저속`, `0~<2 매우저속`
- 압력 라벨: `>=180 높음`, `126~<180 보통`, `<126 낮음`
- 라벨 색상: 상태별 색상이 의도대로 표시되는지 확인

### 2.3 SPOT 경고 검증
- 온도 구간: `480~<540 보통`, `540~<580 고온`, `>=580 경고`, `<480 저온`
- 580 이상에서 경고 테두리/아이콘 표시 확인
- 스파크라인 색상 변화(저온/보통/고온/경고) 확인

### 2.4 카메라 상태 검증
- `2 x refresh_interval` 초과 시 지연 라벨 표시
- `5 x refresh_interval` 초과 시 연결 끊김 표시
- 상태 오버레이 텍스트/스피너 표시 확인

### 2.5 결측/지연/오류 검증
- 결측 시 `--` 표시와 비활성 스타일 적용 확인
- 지연/오류 상태 배지 표시 확인
- 마지막 갱신 시각(`HH:MM:SS`) 표시 확인

### 2.6 편집 모드 검증
- 편집 모드에서 드래그/리사이즈 가능 확인
- 카드 최소 크기/텍스트 반응형 유지 확인
- 레이아웃 저장/복구 동작 확인
- 저장 시 이름 입력 UI 동작 확인
- 최대 3개 저장 제한 확인
- 저장 목록에서 선택 복구 확인
- 저장 시 `layout.json` 생성 및 `layout.backup.json` 백업 생성 확인
- 복구 시 `layout.backup.json` 기준으로 복원 확인
- 로컬 스토리지 레이아웃 보유 시 1회 마이그레이션 동작 확인

### 2.7 설정 UI + 핫 리로드 검증
- logpath 변경 → 재시작 없이 적용, 새 경로에 CSV 생성 확인
- logpath 쓰기 불가 → fallback 경로 사용 및 경고 로그 확인
- rotation/autoSave 변경 → 즉시 반영 확인
- SPOT refresh 변경 → `/api/spot/config` 값 변경 확인(프론트는 재조회 필요)
- Extruder/LS IP/Port 변경 → 소켓 재연결 후 데이터 복귀 확인
- config.ini 외부 수정 → watcher가 적용/로그 기록 확인
- 설정 저장 후 적용 결과(applied/pending) 요약 표시 확인

### 2.8 재시작 필요 항목 검증
- V2_MODE 변경 시 재시작 필요 상태 표시 확인
- 임계값(THRESHOLDS) 변경 시 pending 표시 확인

## 3. 완료 기준
- 모든 시나리오가 정상 동작하면 UI 구현 완료로 판단한다.
