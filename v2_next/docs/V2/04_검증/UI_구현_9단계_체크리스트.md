# UI 구현 9단계 체크리스트

## 1. 목표
- 구현 단계별 완료 여부를 점검하기 위한 체크리스트를 제공한다.

## 2. 체크리스트

### 2.1 레이아웃/구조
- [x] 1920x1080 기준 레이아웃 적용
- [x] 3컬럼 구조 고정
- [x] 카드 간격(12~16px) 적용
- [ ] 컬럼 간격(16~20px) 고정 값 적용 확인 필요

### 2.2 카드 구현
- [x] KPI 카드: 속도/압력 라벨 표시 + 얇은 Bar
- [x] SPOT 카드: Arc Gauge + 500°C 경고 강조
- [x] Secondary Temps: 2x2 서브 그리드 적용
- [x] Mold Zones: 2x3 그리드 + 셀 강조
- [x] Environment: 2열 소형 카드 + 상태 배지
- [x] Camera: 연결 상태 텍스트+스피너 오버레이
- [x] Operator Check: 경고 아이콘+조치 문구

### 2.3 상태/임계값
- [x] 2~5초 지속 조건 적용
- [x] 라벨 기반 표현(Normal/Warning/Danger 미사용) 확인
- [x] SPOT 500°C 위험 강조

### 2.4 편집 모드
- [x] 편집 모드에서만 드래그/리사이즈
- [x] 저장 버튼 편집 모드에서만 활성
- [x] 저장 성공 비차단 알림
- [x] 레이아웃 저장이 `layout.json`에 기록됨
- [x] 레이아웃 이름 입력/저장 동작
- [x] 레이아웃 최대 3개 제한 동작
- [x] 저장 목록에서 선택 복구 동작
- [x] 저장 목록에서 삭제 동작
- [ ] 비상 복구 API(`/api/layout/restore`) 확인

### 2.5 결측/지연/오류
- [x] 결측 시 `--` + 회색 처리
- [x] 지연/오류 라벨 + 아이콘 표시
- [x] 카메라 지연/끊김 조건 반영

## 3. 완료 기준
- 모든 항목이 체크되면 UI 구현 단계 완료로 간주한다.
