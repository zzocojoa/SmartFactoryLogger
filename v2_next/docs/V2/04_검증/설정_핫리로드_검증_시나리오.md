# 설정 핫리로드 검증 시나리오

## 1. 목적
- 즉시/자동 설정 변경이 실제 동작에 반영되는지 검증
- Extruder/LS/Spot/Logger 변경 경로를 분리해 증빙 로그를 남김

## 2. 실행 환경
- 개발 PC(MOCK) + 현장 PC(REAL) 병행 검증
- 공통 환경 변수: `SFL_CONFIG_PATH`, `V2_MODE`

## 3. 검증 범위
- Extruder: IP/Port 변경 핫리로드
- LS PLC: IP/Port + 레지스터 매핑 변경 핫리로드
- SPOT: URL/Refresh/오버레이 파라미터 핫리로드
- Logger: logpath/autosave/rotation/header 핫리로드

## 4. 공통 준비 절차
1) 백엔드 실행(현장: REAL / 개발: MOCK)
2) `/health` 확인(드라이버 상태)
3) `system.log` 및 `comm_metrics.log` 위치 확인

## 5. 시나리오별 검증 절차

### 5.1 Extruder 핫리로드
- 변경 항목: `EXTRUDER.ip`, `EXTRUDER.port`
- 절차:
  1) `config.ini`에서 IP/Port 변경
  2) `system.log`의 적용 로그 확인
  3) `comm_metrics.log`의 `EX_DISCONNECTED` / `EX_RECOVERED` 이벤트 확인
  4) `/health`의 comm metrics로 연결 상태 복구 확인
- 기대 결과:
  - 연결 재시도가 실행되고 복구 시간/횟수가 기록됨
- 실행 상태: 현장 테스트 필요(REAL)

### 5.2 LS PLC 핫리로드
- 변경 항목: `LS_PLC.ip`, `LS_PLC.port`, `LS_PLC_TARGETS`
- 절차:
  1) `config.ini`에서 IP/Port 변경
  2) `system.log` applied 기록 확인
  3) `comm_metrics.log`의 `LS_DISCONNECTED` / `LS_RECOVERED` 이벤트 확인
  4) 매핑 변경 후 다음 read 사이클 반영 확인
- 기대 결과:
  - 연결/매핑 적용이 정상 반영됨
- 실행 상태: 현장 테스트 필요(REAL)

### 5.3 SPOT 핫리로드
- 변경 항목: `SPOT.ip`, `SPOT.refreshinterval`, `SPOT.imageurl`, 오버레이 파라미터
- 절차:
  1) `refreshinterval` 변경 → `/api/spot/config` 값 확인
  2) `imageurl`을 오류 URL로 변경 → 오류 상태 표시 확인
  3) 정상 URL로 복구 → 상태 정상 복귀 확인
- 기대 결과:
  - 백엔드는 즉시 적용, 프론트엔드는 다음 주기 반영
- 실행 상태: 개발/현장 공통(부분 검증 가능)

### 5.4 Logger 핫리로드
- 변경 항목: `SETTINGS.logpath`, `SETTINGS.autosave`, `LOGGING.rotationmode`,
  `LOGGING.cycleidletime`, `LOGGING.cyclethresholdpress`, `HEADERS.csv`
- 절차:
  1) `logpath` 변경 → 경로 로그/CSV 생성 확인
  2) `autosave`/`rotationmode` 변경 → 즉시 반영 로그 확인
  3) `HEADERS.csv` 변경 → 다음 파일 생성 시 헤더 반영 확인
- 기대 결과:
  - 파일 핸들 교체 및 적용 로그 기록
- 실행 상태: 개발/현장 공통(부분 검증 가능)

## 6. 실행 결과 요약(2026-01-03, 개발 PC / MOCK)
- 공통: `/health` 모드=MOCK, `system.log`에서 hot reload applied 로그 확인
- Extruder: IP/Port 변경 적용 확인(`/api/config` 업데이트, applied=2). 연결 이벤트는 MOCK라 미발생 → REAL 필요
- LS PLC: IP/Port 변경 적용(applied=2), LS_PLC_TARGETS 1건 변경 적용(applied=1). 연결/매핑 실기기 검증 필요
- SPOT: refreshinterval 변경 → `/api/spot/config` 반영(applied=1). imageurl 오류/복구는 미수행(화면/장비 필요)
- Logger: logpath/ autosave/ rotation/ cycle 값 변경 적용(applied=6). CSV 생성/헤더 반영은 실데이터 필요
- 특이사항: 초기 테스트 중 config 편집 오류 1회 기록(`Config watch failed`), 백업 복구 후 정상 반영
