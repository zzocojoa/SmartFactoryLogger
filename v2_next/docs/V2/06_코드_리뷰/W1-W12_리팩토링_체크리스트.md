# W1-W12 리팩토링 체크리스트

- 범위: W1~W12 총 167개 파일
- 목표: 동작 불변(behavior-preserving) 구조 개선
- 원칙: UI/로직/상수/유틸/훅/스타일/타입/서비스 분리

## 의존성 방향

`frontend: types/constants/utils -> api/timeseries -> hooks/context -> components -> pages/scenes -> app entry`

`backend: models/config -> services -> routers/app -> scripts`

## 웨이브 게이트

- W1: 순수 함수 우선, 부수효과 금지
- W2: API/워커 계약(시그니처/메시지) 불변
- W3: 타이머/네트워크/구독 효과를 hook/service로 격리
- W4: 컴포넌트는 렌더링 중심으로 슬림화
- W5: 페이지/씬은 오케스트레이션 중심
- W6: 스타일/선언 파일 정합성 유지
- W7~W9: backend 계층 분리(계약 -> 서비스 -> 라우터/엔트리)
- W10~W12: 스크립트는 진입점과 공통 실행 로직 분리
