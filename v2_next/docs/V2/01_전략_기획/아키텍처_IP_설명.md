# V2.0 아키텍처: "IP를 사용한다"는 의미의 정확한 이해

`작성일: 2025-12-31`
`주제: 웹 기술(웹 기술) 도입 시 네트워크/IP 의존성 오해 풀기`

## 1. 질문의 핵심과 답변

**Q: "그렇다면 IP를 사용한다는 거지?" (인터넷이 필요하거나 복잡한 네트워크
설정이 필요한가?)**

**A: "반은 맞고 반은 틀립니다."** 정확히 말하면 **"내부 루프백 주소 (Localhost,
127.0.0.1)"**를 사용합니다. 하지만 사용자 입장에서는 **IP를 몰라도 되고, 인터넷
선을 뽑아도 작동**합니다.

---

## 2. 작동 원리 (내부 vs 외부)

### 2.1. 기본 모드: "인터넷 없는 독립 실행" (현재와 동일)

공장 PC 한 대에서 프로그램이 돌아가는 방식입니다.

- **구조**: 프로그램 내부적으로 `127.0.0.1` (나 자신)이라는 주소로 통신합니다.
- **체감**: 사용자 눈에는 그냥 **`.exe` 프로그램**입니다.
- **IP 설정**: **필요 없음**. (랜선 뽑아도 됨)

### 2.2. 확장 모드: "원격 모니터링" (V2의 장점)

만약 공장장님이 "내 사무실 컴퓨터나 스마트폰에서 현황을 보고 싶다"고 하면, 그때
**IP**가 등장합니다.

- **구조**: 현장 PC의 IP (예: `192.168.0.10`)를 주소창에 치면 접속됩니다.
- **IP 설정**: **필요함**. (이 기능을 쓸 때만)

---

## 3. 요약 비교

| 구분             | V1 (Tkinter)               | V2 (웹 기술)                                           |
| :--------------- | :------------------------- | :------------------------------------------------------ |
| **실행 위치**    | 오직 그 PC 화면에서만 보임 | 그 PC 화면 + (원하면) 스마트폰, 옆자리 PC               |
| **통신 방식**    | 내부 메모리 직접 참조      | 내부 네트워크 통신 (`localhost`)                        |
| **인터넷 필요?** | X (불필요)                 | **X (불필요 - 똑같습니다)**                             |
| **IP 관리**      | 필요 없음                  | **평소엔 필요 없음** (원격 접속 할 때만 IP 알려주면 됨) |


### 3.1. 2025 실무 보완 포인트
- 기본 바인딩은 127.0.0.1로 고정하고, 원격 모드에서만 0.0.0.0으로 확장.
- 원격 모드 시 인증/권한, TLS, IP 허용 목록, 방화벽 규칙을 필수로 적용.
- 현장 안정성을 위해 백엔드는 서비스로 등록하고 자동 재시작/헬스체크를 제공.
- 원격 접근은 VPN/사내망 경유 원칙, 공용 인터넷 직노출 금지.
- 포트/접속 모드는 config로 제어하고 UI에서 현재 모드를 명확히 표시.
- 포트 충돌 감지와 사용자 안내(대체 포트 제안/재시도) 로직을 포함한다.
- 현장 방화벽/보안 정책에 맞춘 예외 등록 가이드와 사전 체크리스트를 제공한다.

## 4. 결론

**"IP를 쓴다는 것 때문에 겁먹으실 필요 없습니다."**

개발자 내부적으로는 IP 프로토콜을 쓰지만, **현장 작업자** 입장에서는 V1과 똑같이
**"그냥 아이콘 더블 클릭하면 켜지는 프로그램"**일 뿐입니다. 단지, **"이제
스마트폰으로도 볼 수 있어요"**라는 옵션이 하나 더 생기는 것뿐입니다.
