# V2.0 (웹 기술) 상용 배포 가이드: 2025 실무 표준

`작성일: 2025-12-31`
`주제: 개발용 PC(Windows)에서 만든 Web Stack 프로그램을 현장 PC(Windows)에 이식하는 법`

## 1. 질문에 대한 직답

**Q: "이것도 빌드를 하는 것인가?"** **A: "네, 무조건 합니다. 심지어 V1보다 더
철저하게 합니다."**

웹 기술(React, Vue)는 소스 코드를 그대로 배포하지 않습니다. 공장 PC 인터넷이 안
되어도 더블 클릭 한 번으로 설치되도록 **"설치 파일(.exe / .msi)"로 만드는
과정(패키징/빌드)**이 필수입니다.

---

## 2. 2025년 실무 프로세스 (Build & Deploy Workflow)

개발자 PC에서 인터넷이 없는 현장 PC까지 가는 표준 여정을 정리해 드립니다.

### Step 1. 빌드 (Build) - 개발자 PC에서 수행

우리가 작성한 수천 개의 코드 파일(React, Python, CSS 등)을 **하나의 설치
파일**로 압축하는 과정입니다.

1. **프론트엔드 빌드**: `npm run build`
   - React 소스(사람이 읽는 코드)를 브라우저가 이해하는 최적화된
     파일(HTML/JS/CSS)로 변환합니다.
2. **백엔드 패키징**: `pyinstaller` (V1과 동일)
   - Python 로직을 실행 파일로 만듭니다.
3. **통합 패키징 (Bundling)**: `electron-builder` **(핵심)**
   - 위의 1번(React) + 2번(Python) + 크롬 브라우저 엔진(Electron)을 몽땅 묶어서
     **`SmartFactorySetup_v2.0.0.exe`** 라는 설치 파일을 생성합니다.

### Step 2. 이식 (Transfer) - USB 또는 사내망 이용

- **결과물**: `SmartFactorySetup_v2.0.0.exe` (약 100MB~200MB 예상)
- 이 파일 하나만 USB에 담아서 현장으로 가져갑니다.
- 소스 코드 폴더를 통째로 복사하는 방식(X) -> **설치 파일만 복사하는 방식(O)**

### Step 3. 배포 (Deployment) - 현장 PC에서 수행

1. 현장 PC 바탕화면에 `SmartFactorySetup_v2.0.0.exe`를 놓습니다.
2. **더블 클릭**합니다.
3. 카카오톡이나 엑셀 설치하듯이 **"설치하시겠습니까? -> 다음 -> 완료"** 과정이
   진행됩니다.
4. 바탕화면에 아이콘이 생기고, 실행하면 깔끔하게 프로그램이 뜹니다.

## 3. 왜 이렇게 하나요? (소스 복사 vs 빌드 배포)

| 구분         | 소스 코드 통째로 복사 (옛날 방식)                       | 설치 파일 빌드 배포 (2025 표준)                               |
| :----------- | :------------------------------------------------------ | :------------------------------------------------------------ |
| **보안**     | 현장 PC에서 소스 코드를 누구나 볼 수 있음 (유출 위험)   | **코드가 암호화/압축되어 있어 안전함**                        |
| **안정성**   | "내 컴퓨터에선 되는데 현장에선 안 돼요" 발생 빈번       | **내 컴퓨터 환경을 그대로 박제해서 가져가므로 100% 동일작동** |
| **편의성**   | Python 깔고, 라이브러리(`pip install`) 깔고 설정해야 함 | **그냥 `Setup.exe` 더블 클릭이면 끝 (Python 안 깔아도 됨)**   |
| **업데이트** | 파일 하나하나 덮어씌우다가 실수할 수 있음               | **새버전 `Setup.exe` 덮어쓰면 자동 업데이트 완료**            |


### 3.1. 2025 실무 보완 포인트
- 설치 파일 코드 서명 및 해시 제공(SHA-256)으로 위변조 방지.
- Webview2 런타임 포함 또는 설치 체크.
- 업데이트 시 config.ini/로그/CSV는 ProgramData 또는 사용자 폴더에 분리 저장하여 덮어쓰기 방지.
- 무인 설치 옵션(/S)과 롤백 패키지 준비.
- 현장 NAS 경로/드라이브 매핑 검증 후 설치 완료 처리.
- 설치/업데이트 시 기존 config.ini 및 사용자 데이터 백업/복원 절차를 기본 제공한다.
- 언인스톨 시 사용자 데이터 보존 여부를 선택 가능하게 설계한다.

현장에 가서 배포하는 작업은 **"USB 꽂고 -> 설치 파일 더블 클릭"**으로 끝납니다.
