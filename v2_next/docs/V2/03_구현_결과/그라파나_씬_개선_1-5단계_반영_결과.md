# V2 그라파나 씬 개선 단계 1-5 반영 결과

## 1. 작업 범위
- 레이아웃 저장/복원 id 기반 전환
- 레이아웃 저장 로직의 상태 구독 방식 전환
- Scenes Runtime 초기화 가드 적용
- 한글 문자열 인코딩 정규화 및 UI 텍스트 분리
- DataContext 타입 정의, 미사용 코드/임포트 정리, 저장 알림 개선

## 2. 반영 내용 요약
1) 레이아웃 저장/복원 방식 개선
- 저장을 인덱스가 아닌 `key` 기반으로 변경하여 카드 순서/추가에 안전하도록 개선.
- 기존 배열 형식 저장본은 로딩 시 id 매핑으로 자동 이관.
- 관련 파일: `v2_next/frontend/src/scenes/DashboardScene.ts`, `v2_next/frontend/src/App.tsx`

2) 레이아웃 저장 로직 안정화
- `grid.state` 직접 접근 대신 `SceneGridLayout` 상태 구독으로 최신 레이아웃 맵을 유지.
- 저장 시 레이아웃 캐시를 사용해 동작 일관성 확보.
- 관련 파일: `v2_next/frontend/src/App.tsx`

3) Scenes Runtime 초기화 가드
- 모듈 최상단 실행 제거, `useEffect`에서 1회 실행되도록 보호.
- HMR/StrictMode 중복 초기화 가능성 감소.
- 관련 파일: `v2_next/frontend/src/App.tsx`

4) 인코딩 깨짐 해결 + 텍스트 분리
- 공지/헤더 한글 문자열을 정상화하고 `uiText.ts`로 분리.
- SPOT 단위 문자열도 상수화하여 재발 방지.
- 관련 파일: `v2_next/frontend/src/constants/uiText.ts`, `v2_next/frontend/src/App.tsx`

5) 타입/알림/정리 개선
- `DataContext` 타입 정의로 안전성 강화.
- 미사용 렌더러 블록 제거 및 불필요한 import 정리.
- 레이아웃 저장 알림은 `alert` 대신 비동기 메시지로 변경.
- 관련 파일: `v2_next/frontend/src/App.tsx`, `v2_next/frontend/src/scenes/DashboardScene.ts`

## 3. 반영된 파일 목록
- `v2_next/frontend/src/App.tsx`
- `v2_next/frontend/src/scenes/DashboardScene.ts`
- `v2_next/frontend/src/constants/uiText.ts`

- 관련 파일: `v2_next/frontend/src/scenes/ScenesRuntime.tsx`
