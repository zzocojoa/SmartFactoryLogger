# 현재 구현 현황 및 미반영 항목

## 1. 목적
- V2 구현 상태를 기준으로 반영된 로직과 미반영/보완 항목을 정리한다.

## 2. 구현 완료(UI)
- 레이아웃/편집 모드: 60x20 그리드 + 4px 보조, 드래그/리사이즈, 저장/복구/삭제(슬롯 기반)
- 레이아웃 저장/복구: 서버/파일 기반 저장/복원 + 슬롯 3개/이름 지정 + 삭제, 저장 시 최신 크기 반영
- 그리드 스냅: Scenes 그리드 상수(60/20/4) 맞춤 패치 적용으로 가로/세로 스냅 일치
- KPI 카드: 속도/압력 상태 라벨, 게이지, 카운트/종료 위치 표시, 정체(0속도+압력) 경고
- SPOT 온도: 4단계(저온/보통/고온/경고), Arc 게이지, 경고 아이콘
- SPOT 스파크라인: 최근 10포인트, 경계선, 상태별 색상
- SPOT 스파크라인 임계값 라인: 설정 임계값 연동 표시
- 온도/환경 카드: 타일화 레이아웃, 상태 색상, 결측 표시
- Mold 카드: Mold1~6 표시, 임계 강조, 결측 표시
- 카메라 카드: 이미지/로딩/지연/오류 오버레이, 크로스헤어, 포커스 버튼
- 반응형: 카드 컨테이너 쿼리 기반 폰트/간격/레이아웃 반응형
- 설정 UI: 요약/섹션 네비게이션, 변경 추적, 유효성 검사, 연결 테스트,
  경로 검사, 로그 회전/알림 섹션 확장, 오버라이드/읽기전용 처리,
  기본값/백업 복원, 임계값 설정, 적용 결과 표시
- 설정 변경 감지: 외부 변경 알림/수동 반영, 보류된 설정 저장(적용/삭제) 표시
- 상태 배지 고도화: 통신 상태/최근 오류/윈도 지표 기반 Warning 표시, 진단/내보내기 버튼
- 운영/관측성 섹션: 윈도 지표/에러 큐 상세/내보내기 경로/브라우저 오류 큐 표시

## 3. 구현 완료(백엔드/운영)
- FastAPI 서비스 구조 + PLC 서비스 스레드 기반 수집
- Real/Mock 드라이버 전환(`V2_MODE`)
- config.ini 로딩/인코딩 대응 + 환경 변수 오버라이드
- config.ini 마이그레이션: legacy 경로 → AppData 표준 경로 자동 복제 + .bak 생성
- 설정 저장/백업/복원 + 로컬 오버라이드 권한 관리
- 설정 저장 안전성: UTF-8 BOM 저장 + 실패 시 `config.pending.json` 기록
- 보류 설정 처리: pending 적용/삭제 API + 성공 시 pending 정리
- 운영 관측성 지표: `/stats` 윈도 지표 + 에러 큐 + 내보내기(스냅샷 파일)
- 관측성 내보내기 열기: 내보낸 파일/폴더 오픈 API
- 내보내기 경로 복구: 최근 내보내기 경로/파일을 재시작 후에도 조회 가능
- 브라우저 오류 포함: 내보내기 스냅샷에 프론트 오류 큐 포함
- 통신 복구 지표화: 재연결/복구 시간/복구 횟수/다운타임(현재·누적) 지표 제공
- Windows 자동 실행 스크립트: 작업 스케줄러 등록/삭제 + 실행 스크립트 제공
- 중앙 설정 서비스: 최신 조회/업데이트/히스토리/롤백/ACK API(파일 저장 기반)
- Windows 서비스 등록 스크립트: 서비스 설치/제거 + 러너 스크립트 제공
- 중앙 설정 승인 흐름: 변경 요청/승인/거절 + 정책 조회/수정 + 간이 UI
- 중앙 설정 저장소: JSON/SQLite 모드 지원(환경 변수로 전환)
- 서비스 안전 종료: `/api/control/shutdown` + 종료 스크립트 제공
  - 스크립트: `v2_next/scripts/stop_windows_service.ps1`
- 레이아웃 저장/복구/삭제 API + 파일 저장(`layout.json`/`layout.backup.json`)
- 프론트 의존성 충돌 해소: `@types/node` 22.12+로 상향, `npm install` 정상화
- 설정 핫 리로드(부분 적용)
  - 로그/저장: logpath/snapshotpath/autosave/rotation/CSV 헤더 즉시 적용
  - SPOT: IP/URL/refresh/crosshair/위젯/액추에이터 파라미터 즉시 적용
  - PLC 통신: Extruder/LS IP·Port 변경 시 소켓 재연결
  - 수집 주기: 정책상 0.2초 고정 적용
- config.ini 변경 감지(watcher) + 적용 결과 로그 기록(system.log)
- 적용 결과(applied/pending) 응답 반영

## 4. 부분 반영/제약

### 개발 보완 필요
- 설정 요약: 최근 저장 시각 표시 미반영
- 보안 섹션: 비밀번호 상태 배지 미반영(placeholder만 표시)
- Scenes 그리드 상수 패치는 `postinstall`로 자동 재적용(스크립트: `v2_next/frontend/scripts/patch_grafana_scenes_grid.js`)
- 중앙 설정 서비스: API 키 기반 RBAC만 지원(사용자/조직 관리 기능 미구현)
- Windows 서비스: 관리자 권한 필요, 서비스 제어는 스크립트 기반

### 현장 테스트 필요
- SPOT 포커스/액추에이터: API 연결은 완료, 실제 동작 검증 미완
- 데이터 정합성: V1 vs V2 수치 검증 절차는 문서화됐으나 현장 검증 미완
- 임계값(thresholds): 백엔드/프론트 적용 완료, 현장 임계값 검증 미완

### 정책/운영 제한
- 수집 주기: 정책상 0.2초로만 적용(설정 변경은 무시됨)
