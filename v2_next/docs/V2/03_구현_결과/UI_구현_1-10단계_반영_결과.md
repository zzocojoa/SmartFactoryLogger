# UI 구현 1-10단계 반영 결과

## 1. 목적
- UI 구현 1~10단계 요구사항이 실제 코드에 반영된 범위를 정리한다.

## 2. 반영 범위
- `v2_next/frontend/src/App.tsx`
- `v2_next/frontend/src/App.css`
- `v2_next/frontend/src/scenes/DashboardScene.ts`
- `v2_next/frontend/src/scenes/ReactWidgetObject.tsx`

## 3. 단계별 반영 요약

### 3.1 레이아웃/구조
- 1920x1080 기준 3컬럼 레이아웃 기본 배치 반영.
- 편집 모드에서만 그리드 배경(60열/20px/4px) 표시.
- 레이아웃 저장/복원은 서버/파일 기반 + 최대 3개 슬롯 저장/선택 복원.

### 3.2 카드 구현
- KPI: 속도/압력 라벨 + 얇은 가로 Bar 적용, 보조 KPI(카운트/종료 위치) 분리.
- SPOT: Arc Gauge + 500°C 이상 정적 테두리 + 경고 아이콘 적용.
- Secondary Temps: 2x2 서브 그리드 적용.
- Mold Zones: 2x3 그리드 + 셀 단위 강조(>=100).
- Environment: 2열 배치 + 상태 배지(온도/습도 라벨).
- Camera: 연결/지연/오류 상태 오버레이 + 스피너 적용.
- Operator Check: 경고 아이콘 + 동적 메시지 노출.

### 3.3 상태/임계값
- KPI 정지/막힘: `speed==0 && press>=20` 조건 2초 경고, 5초 위험.
- SPOT 500°C 위험 조건 2초 지속 적용.
- Secondary Temps/빌렛 온도 경고/위험 조건 2초 지속 적용.

### 3.4 편집 모드 정책
- 운영 모드: 드래그/리사이즈 비활성.
- 편집 모드: 헤더 드래그 허용, 우하단 리사이즈 핸들만 노출.
- 저장 성공 시 비차단 메시지, 실패 시 오류 라벨/재시도 버튼 표시.

### 3.5 결측/지연/오류 규칙
- 결측 값은 `--` 표기 + 회색 톤 유지.
- 마지막 갱신 시각(`HH:MM:SS`)을 카드 하단에 표시.
- 카메라 지연/오류는 Warning/Danger/Loading/Error 오버레이 분리.

### 3.6 스타일/토큰
- 공통 디자인 토큰(`--bg-main`, `--state-*`, `--font-*`) 적용.
- 카드/라벨/상태 클래스는 토큰 기반으로 정리.

## 4. 확인 필요 사항
- 컬럼 간격 16~20px 고정 값은 Scenes 기본 margin(4px) 기반이라 추가 확인 필요.
- `SceneGridLayout` 특성상 `preventCollision`, `compactType` 같은 RGL 옵션은 직접 제어 불가.

## 5. 결론
- 1~10단계 요구사항의 UI 핵심 요소는 대부분 반영 완료.
- 추가 조정이 필요한 부분은 컬럼 간격 규정과 Scenes 레이아웃 제약 검토이다.
