# 설정 핫 리로드 반영 현황

## 범위
- 대상: V2 백엔드 설정 핫 리로드(재시작 없이 적용)
- 기준: config.ini 변경 후 즉시 적용 여부

## 반영됨
- 런타임 설정 로더/차이(diff) 계산
  - config.ini 변경 시 변경 키를 비교해 적용/보류 목록 생성
- 로그/저장 관련 핫 리로드
  - logpath, autosave, rotation_mode, cycle_idle_time, cycle_threshold_press, csv_header
  - 변경 시 파일 핸들 재오픈 및 즉시 적용
- 스냅샷 경로 핫 리로드
  - snapshotpath 변경 즉시 반영
- 수집 루프 간격 핫 리로드(정책 고정)
  - 인터벌은 0.2s 고정 정책 유지, 적용 시에도 0.2로 재설정
- SPOT 설정 핫 리로드
  - ip, url, image_url, refresh_interval
  - crosshair/위젯 크기, focus 관련 파라미터
  - actuator ip/step/url
- PLC 통신 설정 핫 리로드
  - Extruder/LS PLC IP/Port 변경 시 소켓을 닫고 다음 루프에 재연결
  - LS 타깃 목록 변경 반영
- 설정 파일 변경 감지(watcher)
  - config.ini 외부 변경 감지 후 자동 적용
- 감사 로그 기록
  - system.log에 applied/pending 기록

## 부분 반영/제약
- 수집 주기(intervalsec)
  - 설정 변경을 받아도 실제 적용 값은 정책상 0.2로 고정
- SPOT 갱신 주기
  - 백엔드는 즉시 적용되지만, 프론트는 /api/spot/config 재호출이 필요
- LS_TARGETS 변경
  - 적용 후 다음 read 사이클부터 반영됨(즉시 데이터 포맷 변경 가능)

## 설정 변경 후 재시작 필요 여부(현재 로직 기준)

### 재시작 불필요(즉시 적용)
- 저장/로그
  - SETTINGS.logpath, SETTINGS.snapshotpath, SETTINGS.autosave
  - LOGGING.rotationmode, LOGGING.cycleidletime, LOGGING.cyclethresholdpress
  - HEADERS.csv
  - logpath 변경 시 파일 핸들 재오픈(쓰기 불가 시 기본 경로로 fallback)
- SPOT
  - ip, refreshinterval, imageurl
  - focusurl, focusstep, actuatorip, actuatorstep, actuatorurl
  - crosshairx/y/color/thickness/size/gap, widgetwidth/height
- PLC 통신
  - EXTRUDER ip/port, LS_PLC ip/port
  - LS_PLC_TARGETS (다음 read 사이클부터 적용)
- 수집 주기
  - SYSTEM.intervalsec (정책상 0.2로 고정 적용)

### 재시작 필요 또는 현재 pending 처리
- THRESHOLDS_VALUE / THRESHOLDS_ENABLE
  - 백엔드 처리 로직이 없어 pending으로 분류됨
- SETTINGS.password 변경으로 password_set 상태가 변하는 경우
  - 실제로는 즉시 적용되지만 현재 pending으로 표시될 수 있음
- V2_MODE(REAL/MOCK) 전환
  - 프로세스 시작 시 결정되므로 재시작 필요
- 프론트 UI 외부 변경
  - config.ini 외부 변경은 UI에 즉시 반영되지 않음(재조회 필요)

## 미반영(재시작 필요 또는 추가 작업 필요)
- 프론트 UI의 실시간 설정 반영
  - settings 외부 변경은 UI에 즉시 반영되지 않음(재조회 필요)
- 임계값(thresholds) 자체의 백엔드 적용
  - 백엔드 처리 로직이 없어 pending으로 분류됨
  - UI 경고는 설정 저장 후 프론트에서 사용 가능(단, 외부 수정은 UI 재조회 필요)
- V2_MODE(REAL/MOCK) 전환
  - 프로세스 시작 시 결정되므로 재시작 필요

## restart_required 판단 기준
- 핫 리로드로 처리된 항목은 applied로 처리
- 미적용 항목이 존재하면 restart_required = True
