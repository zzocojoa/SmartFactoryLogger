# V2.0 Deployment Strategy (2025 Standard)

## 1. 개요 (Overview)

본 문서는 **SmartFactoryLogger V2**의 배포 및 실행 전략을 정의합니다.
V1(Tkinter)과 달리 V2는 **Frontend(React) + Backend(FastAPI)** 구조를 가지므로,
단순 실행 파일(`.exe`) 배포보다 복잡한 환경 설정이 필요합니다.

2025년 최신 실무 관점에서 **"운영 안정성(Stability)"**과 **"배포
편의성(Convenience)"**을 모두 만족하는 2단계 전략을 제안합니다.

---

## 2. 배포 전략 (Deployment Roadmap)

### Phase 1: 스크립트 기반 원클릭 실행 (Immediate)

**현재 상태에서 가장 빠르고 안정적인 방법입니다.** 복잡한 빌드 과정 없이, 소스
코드를 직접 실행하되 **진입점(Entrypoint)**을 자동화하여 사용자 경험을
단순화합니다.

- **구성:** `Backend Launcher (.bat)` + `Frontend Launcher (.bat)`
- **장점:** 소스 수정 즉시 반영 가능, 디버깅 용이.
- **단점:** Python/Node.js 런타임 설치 필요.

### Phase 2: 패키징 기반 단일 실행 파일 (Target)

**상용 소프트웨어 수준의 배포 방식입니다.** 백엔드는 PyInstaller로, 프론트엔드는
Electron으로 감싸 하나의 설치 파일(`.exe` / `.msi`)로 만듭니다.

- **구성:** `Electron (Frontend Shell)` + `PyInstaller (Backend Binary)`
- **장점:** 런타임 불필요, 코드 보안, 깔끔한 설치.
- **단점:** 빌드 파이프라인 구성 비용 발생, 핫픽스 시 재빌드 필요.

---

## 3. Phase 1 상세 구현: 원클릭 런처 (One-Click Launcher)

현장 작업자가 복잡한 명령어를 입력하지 않도록 **배치 파일(.bat)**을 제공합니다.

### 3.1. 디렉토리 구조

```text
C:\Users\user\Desktop\v2_next\
├── backend\
├── frontend\
├── RUN_V2_REAL.bat   <-- [New] 현장용 실행 스크립트
├── RUN_V2_MOCK.bat   <-- [New] 테스트용 실행 스크립트
└── UPDATE.bat        <-- [New] NAS 동기화 스크립트
```

### 3.2. `RUN_V2_REAL.bat` (작성 예시)

이 스크립트는 다음 작업을 순차적으로 수행합니다.

1. **환경 변수 설정**: `V2_MODE=REAL`, `SFL_CONFIG_PATH` 등.
2. **백엔드 실행**: 별도 창(Hidden/Minimized)에서 Python 서버 시작.
3. **프론트엔드 실행**: `npm start` 또는 빌드된 정적 파일
   서빙(`serve -s build`).
4. **헬스 체크**: 백엔드 구동 대기 후 브라우저 오픈.

### 3.3. `UPDATE.bat` (유지보수)

NAS(`Z:`)에서 최신 소스를 `Robocopy`로 안전하게 가져오는 스크립트입니다.

## 4. Phase 2 상세 가이드: Electron 패키징 (Future)

추후 도입할 **Electron** 패키징 구조는 다음과 같습니다.

1. **Frontend Build**: `npm run build` -> `build/` 폴더 생성.
2. **Backend Build**: `pyinstaller --onefile backend/main.py` -> `backend.exe`
   생성.
3. **Electron Builder**:
   - `build/` 폴더를 정적 리소스로 포함.
   - `backend.exe`를 `extraResources`로 포함.
   - 앱 시작 시 Electron 메인 프로세스가 `backend.exe`를 자식 프로세스로
     실행(Spawn).
