# MES 데이터 수집 워크플로우 (Logic Summary)

MES 시스템에서 데이터를 추출하여 JSON으로 저장하는 핵심 프로세스는 총 5단계로 요약됩니다.

## 1. 초기화 및 설정 (Setup)

- **정보 로드**: `config.ini`에서 계정 정보(ID/PW)를 읽고, `page_structures.json`에서 수집 대상 페이지명, URL, 테이블 ID, 필터 유형을 로드합니다.
- **환경 준비**: `Playwright` 브라우저 인스턴스를 초기화하고 로그인 세션을 확보합니다.

## 2. 페이지 진입 및 필터링 (Navigate & Filter)

- **URL 이동**: 대상 페이지로 이동 후 `networkidle` 상태까지 대기합니다.
- **날짜/연도 설정**:
  - `date_range`: 시작/종료 입력 필드에 `JavaScript`로 값을 주입하고 검색 버튼을 클릭합니다.
  - `year`: 연도 선택 `Select` 박스 옵션을 변경합니다.
  - `search`: 필터 없이 검색 버튼만 클릭하여 마스터 데이터를 로드합니다.

## 3. 데이터 추출 (Data Extraction - 핵심)

- **JS Evaluate**: 브라우저 컨텍스트 내에서 `JavaScript`를 실행하여 DOM 테이블을 파싱합니다.
  - `gv_Board` 등의 테이블 ID를 타겟팅합니다.
  - 첫 행에서 헤더(컬럼명)를 추출합니다.
  - 나머지 행을 순회하며 '합계', '소계' 행은 제외하고 데이터를 `List[Dict]` 형태로 변환합니다.

## 4. 페이지네이션 처리 (Pagination)

- **반복 추출**: 현재 페이지 데이터를 수집한 후 '다음(Next)' 버튼의 존재 여부와 활성화 상태를 체크합니다.
- **클릭 및 대기**: 버튼이 있으면 클릭하고 데이터가 로딩될 때까지 대기(Wait) 후 3번 과정을 반복합니다.

## 5. 정제 및 JSON 저장 (Save)

- **데이터 패키징**: 추출 시간, 연도, 수집 결과, 에러 메시지를 포함한 메타데이터와 함께 최종 데이터를 구성합니다.
- **디렉토리 관리**: `data/[카테고리]/[페이지명]/` 구조로 폴더를 자동 생성합니다.
- **저장**: 완성된 객체를 `json.dump`를 통해 한글 인코딩(`ensure_ascii=False`)을 유지하며 저장합니다.

---

### 💡 핵심 기술 포인트 (Stable Scraping)

- **Timeout 관리**: 서버 응답 속도에 따라 페이지 이동 및 검색 대기 시간을 탄력적으로 조정(15s~60s).
- **에러 핸들링**: 특정 페이지 실패가 전체 프로세스를 중단시키지 않도록 `Try-Except` 블록으로 개별 처리.
- **동적 로딩 대응**: 단순 `goto`가 아닌 `wait_for_load_state`와 필요 시 `asyncio.sleep`을 혼합하여 안정성 확보.
