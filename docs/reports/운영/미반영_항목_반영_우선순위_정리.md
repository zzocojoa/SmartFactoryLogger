# 미반영 항목 반영 우선순위 정리

## 정리 기준
- "미반영/부분 반영/확인 필요" 항목을 "꼭 반영(필수)"과 "보류 가능(권장/조건부)"으로 구분했습니다.

## 꼭 반영해야 할 내용 (필수)

| 항목 | 반영 사유 | 미반영 허용 사유 |
| --- | --- | --- |
| 로그 경로가 유효하지 않으면 `APP_DATA_DIR/logs`로 자동 전환합니다. (미반영) | 운영 안정성/데이터 무결성 | 없음 |
| 단일 진입점: `config.ini`만 수정, 화면 변경은 파일에 저장. (화면 확인 필요) | 설정 일관성/오류 예방 | 없음 |
| 안전 저장: 임시 파일 작성 후 원자적 교체, `config.ini.bak` 순환 백업 유지. (부분 반영) | 설정 손상 방지/데이터 무결성 | 없음 |
| 자동 전환: 로그/스냅샷 경로가 유효하지 않으면 `APP_DATA_DIR`로 전환, 경고 로그 기록. (미반영) | 운영 안정성/데이터 무결성 | 없음 |
| 읽기 전용: `PermissionError` 발생 시 “관리자 권한이 필요합니다” 같은 명확한 화면 안내를 제공하고, 저장 실패 시 임시 경로 저장/재시도 정책을 둔다. (미반영) | 운영 안정성/데이터 무결성 | 없음 |
| 원자적 파일 입출력 및 백업/롤백 전략. (부분 반영) | 설정 손상 방지/데이터 무결성 | 없음 |
| 설정 실패에 대한 운영 로그/감시. (부분 반영) | 장애 원인 추적 | 없음 |
| 이식 PC `config.ini` 수정 후 반영 확인. (수동 확인) | 현장 적용 검증 | 없음 |
| 잘못된 타입/범위 주입 후 기본값 보정과 경고 로그 확인. (수동 확인) | 현장 적용 검증 | 없음 |
| 읽기 전용 경로에서 자동 전환 및 데이터 손실 여부 확인. (수동 확인) | 현장 적용 검증 | 없음 |

## 보류 가능/조건부 항목 (권장/조건부)

| 항목 | 반영 사유 | 미반영 허용 사유 |
| --- | --- | --- |
| 상용 배포에서 사용자 설정 값 관리 운영 규칙 정의. (문서) | 운영 기준정렬/인수인계 | 내부 공유 시 보류 가능 |
| 본 프로그램은 개발 환경에서 빌드 후 실제 공장 현장 PC로 이식하여 사용되는 전제를 가집니다. (문서/절차) | 운영 기준정렬/인수인계 | 내부 공유 시 보류 가능 |
| 따라서 `config.ini`와 `src/config.py`의 경로/모드(포터블/표준) 설계 및 변경은 현장 이식/운영 시나리오를 기준으로 검토하는 것이 중요합니다. (문서/절차) | 운영 기준정렬/인수인계 | 내부 공유 시 보류 가능 |
| 현장 PC에는 `config.ini`를 동봉 배포하여 IP/포트 변경을 운영자가 즉시 반영할 수 있도록 하는 것이 적합합니다. (문서/절차) | 운영 기준정렬/인수인계 | 내부 공유 시 보류 가능 |
| 우선순위: 이식 PC `config.ini` > `APP_DATA_DIR` 기본값 > 코드 기본값. (부분 반영) | 운영 효율/현장 대응성 | 운영 영향 적음/문서/절차로 대체 가능 |
| 보안: 가능하면 평문 설정에 인증 정보 저장 금지. (미반영) | 보안 요구 | 보안 요구 적음 |
| 환경 변수: 자동 배포 대비로 환경 변수가 있으면 `config.ini`보다 우선하도록 하고(예: `MYAPP_SERVER_IP`), 덮어쓴 값은 로그로 남긴다. (미반영) | 자동화 대응 | 수동 운영 시 보류 가능 |
| 배포 패키징 및 롤백 절차. (미반영) | 복구 대응 | 수동 운영 시 보류 가능 |
| 동시 실행 시 `app.lock` 잠금 파일로 설정 파일 무결성 확인. (수동 확인) | 동시 실행 위험 | 단일 실행 보장 시 보류 가능 |
| 상용 배포는 설정 검증, 안전 저장, 우선순위 규칙이 필수. (문서 결론) | 운영 기준정렬/인수인계 | 내부 공유 시 보류 가능 |
| 현장 PC의 `config.ini`가 사용자 설정의 단일 기준. (문서 결론) | 운영 기준정렬/인수인계 | 내부 공유 시 보류 가능 |
