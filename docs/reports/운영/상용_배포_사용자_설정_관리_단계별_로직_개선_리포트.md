# 상용 배포 사용자 설정 관리 단계별 로직 개선 리포트

## 1. 목적
- 상용 배포 환경 기준 사용자 설정 관리 로직 단계별 개선.
- 기준 문서: `docs/reports/운영/상용_배포_사용자_설정_관리_리포트.md`

## 2. 전제/운영 기준
- 이식 PC `config.ini` 단일 기준.
- 우선순위: `config.ini` > APP_DATA_DIR 기본값 > 코드 기본값.
- 환경 변수 우선 엎션, 적용 시 로그 기록.

## 3. 단계별 로직 개선
- 단계 0 설정 안전화: UTF-8 with BOM, `utf-8-sig` 우선, `cp949` fallback.
- 단계 1 검증/기본값 보정: `config_schema.py`, 누락 보정, 경고 로그.
- 단계 2 안전 저장/백업: 임시 파일, 원자적 교체, `config.ini.bak`.
- 단계 3 우선순위/환경 변수: ENV 우선순위 문서화, 변경 항목 로그.
- 단계 4 경로/로그 안전화: logpath 검증, `APP_DATA_DIR/logs` 전환, file_writer_thread 곀리.
- 단계 5 중복 실행/보안: `app.lock`, 평문 인증 정보 금지.

## 4. 검증 방법
- 이식 PC `config.ini` 변경/오류 사례 주잎, 반영 홬인.
- 인코딩/권한/ENV/백업 상황 로그 홬인.

## 5. 산출물
- 단계별 점검 목록, 적용 로그, 장애 대응 기록.
