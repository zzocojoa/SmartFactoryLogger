# Result_운영

`작성일: 2025-12-31` `작성자: Antigravity (AI Agent)`

본 문서는 `docs/reports/운영` 폴더 내의 분석 및 보고서를 바탕으로 **수행된
내용**과 **수행되지 않은 내용**을 정리합니다.

## 1. 수행된 내용 (Performed)

### 1.1 사용자 설정(Config) 관리 정책 확립

- **단일 소스 원칙 (Single Source of Truth)**: 현장 이식 PC의 `config.ini`를
  최우선 설정값으로 사용하는 정책 확정.
- **우선순위 로직 적용**: `이식 PC config.ini` > `AppData 기본값` >
  `코드 내 Default 상수` 순서로 로딩되도록 구현.
- **인코딩 호환성 강화**: `UTF-8 with BOM` 생성 및 로딩 시 `utf-8-sig` 우선,
  실패 시 `cp949` Fallback 로직 적용 (한국어 윈도우 메모장 호환).

### 1.2 안전성 및 무결성 확보 기술 적용

- **설정 검증 (Validation)**: `config_schema.py`를 통해 데이터 타입 및 유효 범위
  검증, 누락 값 자동 보정 로직 구현.
- **경로 자동 보정**: `LogPath`, `SnapshotPath` 등의 경로가 유효하지 않을 경우
  `APP_DATA_DIR` 등 안전한 경로로 자동 전환.
- **CSV 헤더 마이그레이션**: 구버전 설정 사용 시 `DIE_ID`, `Billet_CycleID` 등
  신규 필수 컬럼 자동 추가 및 백업 생성 기능 구현.

### 1.3 설정 로직 개선

- **상대 경로 지원**: 설정 파일 내 `./logs`와 같은 상대 경로를 프로그램 실행
  위치 기반 절대 경로로 자동 변환하는 로직 추가.
- **키 정규화**: `LS_PLC_TARGETS` 등 대소문자 혼용 가능성 있는 키를 일괄
  정규화하여 처리.

## 2. 수행되지 않은 내용 / 보류 사항 (Not Performed)

### 2.1 고급 보안 및 운영 기능

- **환경 변수(Environment Variable) 우선 적용**: 컨테이너/자동 배포를 위한 `ENV`
  변수 오버라이드 기능은 현재 수동 운영 환경상 보류됨.
- **평문 비밀번호 암호화**: `config.ini` 내 비밀번호 저장 시 암호화 적용은 보안
  요구사항 수준이 낮아 미반영(평문 저장 유지).
- **읽기 전용(Read-Only) 오류 UI 처리**: `PermissionError` 발생 시 사용자에게
  팝업 등으로 안내하는 UI 기능은 미구현 (로그 기록으로 대체).

### 2.2 운영 절차 문서화 및 자동화

- **배포/롤백 자동화 스크립트**: 패키징 및 버전 롤백을 자동화하는 스크립트
  작성은 보류.
- **현장 운영 매뉴얼 상세화**: "상용 배포 시 사용자 설정 관리 규칙"에 대한
  사용자용 문서는 내부 리포트로만 존재하며, 최종 사용자용 매뉴얼로는 가공되지
  않음.
- **동시 실행 방지 UI 연동**: `app.lock` 파일 기반의 중복 실행 방지 로직은
  있으나, "시작 버튼 비활성화" 등 UI 레벨의 방어 코드는 미반영.
