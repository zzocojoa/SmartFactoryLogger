# 스마트팩토리로거 GUI P1 개선 상세 단계 리포트

## 1. 목적
- P1(중요) 항목을 디자이너 실무 관점에서 단계별 실행안으로 구체화한다.
- 대상: KPI 재배치, 카드 단위/색상 규칙 정규화, 그래프 가독성 개선, 스파크라인.

## 2. 전제 및 설계 기준
- 공장 현장 고정형 화면, 장시간 연속 운영.
- 0.2초 수집 주기에서 지연/끊김을 즉시 인지 가능해야 한다.
- 운영자 3초 내 인지/조치 목표.
- 참고 화면: GUI_dashboard.png (현행 대시보드).

## 3. P1 상세 단계

### 3.1 KPI 재배치
- 1) KPI 재배치: 핵심 지표 1열 배치, 보조 지표는 그룹화.
- 2) 좌측 KPI 컬럼: 게이지를 최상단에 배치하고, Main Pressure → Prod Count → End Position 순으로 평준 정렬한다.
- 3) 시선 유도: 좌상단(헤더/상태) → 중앙 상단(SPOT) → 우측 상단(Mold) → 우측 하단(Pinned Alert) → 중앙 하단(카메라) 순으로 사용자 스캔을 설계한다.

### 3.2 카드 단위/색상 규칙 정규화
- 1) 카드 단위/색상 규칙 정규화
- 2) 상태 기준/표현 규칙: 정상/주의/오류 3단계, 색상+아이콘+텍스트 동시 사용.
- 3) 색상만 의존하지 말고 아이콘/라벨 병행.

### 3.3 그래프 가독성 개선
- 1) 그래프 가독성 개선: 색상 매핑 통일, 임계값 강조.
- 2) 카드 KPI 색상과 그래프 색상 매핑 일치.
- 3) 임계 구간 음영 표시 + 기준선 강조.
- 4) 빠른 시간 범위 프리셋(1~5분, 10~30분 등).

### 3.4 스파크라인
- 1) `InfoCard`에 간소화된 캔버스 추가하여 최근 10개 데이터 점 연결 (Matplotlib 없이 캔버스 라인으로 경량 구현).


### 3.5 디스크 I/O 인디케이터(추가 보완)
- 1) 필요성: P0 구현 결과 보고서 미반영 항목으로 확인된 디스크 I/O 피드백을 P1 단계에서 해결한다.
- 2) 트리거 정의: CSV/로그 저장 시점에 신호를 발생시키고, 저장 실패 시 오류 상태로 전환한다.
- 3) 표현 규칙: 저장 중에는 벨/디스크 아이콘을 300~500ms 점멸 효과로 표시하고, 저장 유휴 시 복원한다.
- 4) 색상 상태: 정상=문제 없음(현행 테마 매칭), 진행=양감 강조, 실패=적색 배지/텍스트로 복합 표시한다.
- 5) 서버/로컬 구분: NAS 저장시와 로컬 저장시를 배지/색상 심볼로 구분하여 운영자가 저장 위치를 직관적으로 파악하게 한다.
- 6) 검증: 저장 실행/실패 시나리오에서 아이콘 점멸 연동 여부를 확인한다.

## 4. 검증 방법
- 3초 인지 테스트(A/B).
- 오류/지연 시나리오 워크스루.
- 현장 운영자 3명 이상 사용성 점검.
