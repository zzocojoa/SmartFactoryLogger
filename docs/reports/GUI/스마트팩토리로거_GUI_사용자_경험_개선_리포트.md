# 스마트팩토리로거 GUI 사용자 경험 개선 리포트

## 1. 목적 및 범위
- 목적: 운영자가 상태/이상 징후를 3초 내에 인지하고 조치할 수 있도록 한다.
- 범위: 대시보드, 그래프 화면, 설정/진단, 알림/상태 피드백.
- 현장 조건: 공장 현장 고정형 디스플레이, 0.2초 수집 주기, 장시간 연속 운영.

## 2. 현행 UX 진단
- 핵심 KPI와 보조 지표의 시각적 위계가 약해 우선순위 판단이 어렵다.
- 상태 라벨/토스트가 일시적이며 심각도/이력 관리가 부족하다.
- 데이터 최신성(지연/정지) 표시가 명확하지 않다.
- 숨겨진 인터랙션(예: 상태 라벨 클릭)으로 발견성이 낮다.

## 3. 디자인 원칙
- 즉시성: 상태/이상/행동이 한눈에 보이도록 한다.
- 일관성: 색상/아이콘/단위/용어의 의미를 화면 전반에서 통일한다.
- 실패 내성: 데이터 누락/지연을 명시적 상태로 보여준다.
- 현장 적합성: 큰 글자, 높은 대비, 최소 조작.

## 4. 개선 백로그 (우선순위)
### P0 (필수)
- 상단 고정 상태 바 추가: 연결 상태, 데이터 최신성, 최근 오류 요약.
- 알림 센터 구성: 심각도/시간/원인/조치 가이드 제공.
- 즉시 조치 버튼 제공: 재연결, 설정 열기, 관리자 호출 등.

### P1 (중요)
- KPI 재배치: 핵심 지표 1열 배치, 보조 지표는 그룹화.
- 카드 단위/색상 규칙 정규화.
- 그래프 가독성 개선: 색상 매핑 통일, 임계값 강조.

### P2 (개선)
- 설정 UX 보강: 입력 검증, 저장 결과, 권한 오류 안내.
- 진단 확장: 장치별 상태, 재시도 횟수, 큐 상태, 마지막 오류 시간.

## 5. 화면별 개선안
### 대시보드
- 데이터 최신성 배지(마지막 업데이트 시각 또는 지연 초).
- 색상만 의존하지 말고 아이콘/라벨 병행.

### 그래프
- 카드 KPI 색상과 그래프 색상 매핑 일치.
- 임계 구간 음영 표시 + 기준선 강조.
- 빠른 시간 범위 프리셋(1~5분, 10~30분 등).

### 설정/진단
- 진단 버튼을 명확히 노출(숨겨진 진입 금지).
- NAS(Z:) 연결 상태를 설정 화면에서 확인 가능하도록 표시.

## 6. 검증 방법
- 3초 내 이상 인지 테스트(A/B 비교).
- 오류/지연 시나리오 기반 조치 명확성 점검.
- 현장 운영자 3명 이상 사용성 점검.

## 7. 담당/ETA (초안)
| 영역 | 담당 | ETA | 검증 |
| --- | --- | --- | --- |
| 상태 바/알림 센터 | UI/UX + UI 개발 | 1.0일 | 인지 시간 테스트 |
| KPI 위계/레이아웃 | UI/UX | 1.0일 | 현장 리뷰 |
| 그래프 가독성 | UI 개발 | 1.0~1.5일 | 임계 인지 정확도 |
| 설정/진단 UX | UI 개발 + 운영 | 1.0일 | 저장/오류 시나리오 |

## 8. 결론
고정 0.2초 수집 환경에서는 정보 위계와 알림 구조가 가장 중요하다. 본 개선안은 가시성, 일관성, 행동 유도에 집중해 장시간 현장 운영에 적합한 UX를 목표로 한다.
