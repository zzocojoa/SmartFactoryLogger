# 스마트팩토리로거 GUI P1 계획 검토 리포트

`작성일: 2025-12-31` `검토자: 시각 디자인 및 UX 전문가`
`검토 대상: 스마트팩토리로거_GUI_P1_개선_상세_단계_리포트.md`

## 1. 개요 (Overview)

본 리포트는 기 작성된 `GUI P1 개선 상세 단계 리포트`에 대한 디자인 실무 관점의
검토 결과입니다. 산업 현장의 특수성(고정형 화면, 즉각적 인지 필요)과 P0 구현
경험을 바탕으로, 계획의 타당성과 구체적인 실행 가이드를 제시합니다.

## 2. 주요 검토 의견 (Key Findings)

### 2.1 KPI 재배치 (Layout efficiency)

- **[긍정]**: 시선 유도 흐름(좌상단 → 우하단)을 명시한 것은 매우 적절합니다.
  'Main Pressure'와 'Prod Count'를 최상단에 배치함으로써 운영자가 가장 빈번하게
  확인하는 정보에 대한 접근성을 높였습니다.
- **[제언]**: 게이지(Gauge) 컴포넌트는 공간을 많이 차지하므로, 단순 나열보다는
  **그룹핑(Grouping)**이 중요합니다. 압력과 속도는 '압출 현황' 그룹으로, 온도
  관련 지표는 '온도 모니터링' 그룹으로 시각적 경계(Border/Bg)를 두어 구분감을
  강화할 것을 권장합니다.

### 2.2 카드 단위/색상 규칙 (Consistency)

- **[긍정]**: '색상+아이콘+텍스트'의 3중 인지 장치는 색각 이상자나 조도 환경이
  나쁜 현장에서 필수적입니다.
- **[제언]**:
  - **Normal (정상)**: 과도한 강조를 피하고, 배경색과 유사한 톤(Dimmed)이나
    중성색(Neutral)을 사용하여 '안정감'을 전달해야 합니다.
  - **Warning (주의)**: 노란색(Yellow/Amber) 계열을 사용하되,
    반짝임(Blink)보다는 **고정된 고채도 색상**을 권장합니다. (반짝임은 피로도
    유발)
  - **Error (오류)**: 적색(Red)과 함께 **굵은 테두리(Stroke)** 또는
    **반전(Invert)** 효과를 고려하십시오.

### 2.3 그래프 가독성 (Readability)

- **[제언]**:
  - **배경 그리드**: 너무 짙은 그리드는 데이터 라인과 간섭을 일으킵니다. 투명도
    10~20% 수준의 얇은 선을 사용하십시오.
  - **범례(Legend)**: 그래프 영역 내부보다는 상단이나 하단에 별도 영역으로
    배치하여 그래프 영역을 최대화하는 것이 좋습니다.
  - **시간 프리셋(Time Presets)**: (추가) '1분', '5분' 등의 버튼은 현장 장갑
    착용 환경을 고려하여 **Touch Target 사이즈(최소 44x44px)**를 확보해야
    합니다. Segmented Control 스타일을 추천합니다.

### 2.4 스파크라인 (Sparkline)

- **[주의]**: `InfoCard` 내부에 캔버스를 추가하는 것은 성능 비용이 큽니다.
  Matplotlib 대신 `tkinter.Canvas`의 `create_line`을 사용하여 **경량화**하는
  것이 핵심입니다. 데이터 포인트는 20개 내외로 제한하여 리소스를 절약해야
  합니다.

### 2.5 디스크 I/O 인디케이터 (Feedback)

- **[긍정]**: P0 검토에서 누락되었던 '저장 피드백'을 P1에 포함시킨 것은 훌륭한
  판단입니다.
- **[구체화 제언]**:
  - **점멸 주기**: 300~500ms는 적절합니다. 너무 빠르면 신경 쓰이고, 너무 느리면
    인지가 어렵습니다.
  - **NAS vs Local**: 아이콘 자체를 변경하기보다는, 네트워크 저장 시 아이콘 옆에
    작은 'Cloud' 또는 'N' 배지를 붙이는 방식을 추천합니다. (예: 💾ᴺ)

## 3. 검증 계획 검토 (Verification Check)

- **3초 인지 테스트**: 매우 실용적인 검증법입니다. 다만, "오류 상황"에서의 인지
  속도와 "정상 상황"에서의 인지 속도를 분리하여 측정해야 합니다. (정상 상황은
  0.5초 내 인지 목표, 오류 상황은 3초 내 구체적 원인 파악 목표)
- **오류 시나리오**: '네트워크 단절' 뿐만 아니라 '데이터 동결(Frozen)',
  '이상치(Outlier) 발생' 시나리오도 포함하여 시각적 피드백(Glow)이 제대로
  작동하는지 확인해야 합니다.

## 4. 종합 결론 (Conclusion)

P1 계획안은 **'정보의 위계 정리'**와 **'직관성 강화'**라는 GUI 개선 목적에
정확히 부합합니다. 특히 **디스크 I/O 인디케이터**의 구체적인 트리거 및 표현 규칙
정의는 운영 신뢰성을 높이는 데 크게 기여할 것입니다.
