# 통신 데이터 요청 최적화 설계 (Data Diet)

`작성일: 2025-12-31`

본 문서는 `Merge Mode` 실패의 근본 원인이 **"비효율적인 대용량 데이터
요청"**임에 착안하여, 필요한 데이터만 핀포인트로 조회하는 **"Mini-Block"**
전략을 설계합니다.

## 1. 현황 및 문제점 분석

### 1.1 현재 방식 (Giant Block)

현재 `_read_merged` 로직은 단 6개의 변수를 얻기 위해 **814개의 워드(Word)**를
요청하고 있습니다.

- **요청 1**: `D0020` ~ `D0421` (402 Words) → 사용: `D23, D31, D32, D421` (4개)
- **요청 2**: `D1500` ~ `D1911` (412 Words) → 사용: `D1510, D1911` (2개)

### 1.2 문제점

- **Payload 낭비**: 실제 유효 데이터 비율이 **0.7%** (6/814)에 불과합니다.
  99.3%는 쓰레기 데이터입니다.
- **장비 부하**: 구형 PLC가 0.2초마다 400워드급 패킷 2개를 처리하고 응답을
  생성하는 것은 과부하를 유발합니다. 이것이 `timeout` 및 `IO Error`의
  주원인입니다.

## 2. 개선 설계 (Smart Mini-Block)

### 2.1 전략: Data Diet

거대한 덩어리를 잘게 쪼개어, 인접한 유효 데이터끼리만 묶어서 요청합니다.

### 2.2 최적화된 블록 구성

전체 요청 크기를 **814 Words → 약 62 Words**로 **92% 이상 감량**합니다.

| Block Name          | Address Range     | Size (Words) | 포함된 주요 변수                             |
| :------------------ | :---------------- | :----------- | :------------------------------------------- |
| **Block 1 (Main)**  | `D0020` ~ `D0035` | 16           | 압력 (`D23`), 온도전 (`D31`), 온도후 (`D32`) |
| **Block 2 (Pos)**   | `D0420` ~ `D0425` | 6            | 종료 위치 (`D421`)                           |
| **Block 3 (Count)** | `D1500` ~ `D1515` | 16           | 생산 카운트 (`D1510`)                        |
| **Block 4 (Bill)**  | `D1900` ~ `D1915` | 16           | 빌렛 길이 (`D1911`)                          |
| **Block 5 (Speed)** | `B1502`           | 1            | 속도 (`B1502`)                               |
| **Total**           |                   | **55 Words** | (기존 대비 1/14 수준)                        |

## 3. 구현 로직 변경 (extruder.py)

### 3.1 `_read_optimized()` 신설

기존 `_read_merged`를 `_read_optimized`로 대체합니다.

```python
def _read_optimized(self):
    # 1. Main Block (Pressure, Temp)
    b1 = self._read_block("D0020", 16)
    if not b1: return None
    
    # 2. End Pos Block
    b2 = self._read_block("D0420", 6)
    if not b2: return None
    
    # 3. Count Block
    b3 = self._read_block("D1500", 16)
    if not b3: return None
    
    # 4. Billet Block
    b4 = self._read_block("D1900", 16)
    if not b4: return None
    
    # 5. Speed (Bit/Word)
    b_spd = self._read_block("B1502", 1)
    if not b_spd: return None
    
    # 매핑 및 리턴 (기존 구조 호환)
    return {
        "Press": b1[3] / 10.0,
        "Temp_F": b1[11],
        "Temp_B": b1[12],
        "EndPos": b2[1] / 10.0,
        "Count": b3[10],
        "Billet": b4[11],
        "Speed": b_spd[0] / 10.0
    }
```

## 4. 기대 효과

1. **통신 안정성 극대화**: 패킷 크기가 작아져서 전송 에러 및 타임아웃 확률이
   획기적으로 낮아짐.
2. **0.2초 주기 완벽 대응**: 가벼운 패킷으로 인해 `RTT(Round Trip Time)`가
   단축되어 고속 폴링 가능.
3. **Merge 실패 제거**: 더 이상 거대 블록을 요청하지 않으므로, `Merge Failed`
   로그가 사라질 것으로 예상됨.
