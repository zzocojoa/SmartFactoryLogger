# 통신 로직 개선 반영 현황 리포트

## 1. 범위
- 대상 문서: `docs/reports/통신/통신_연결_끊김_분석_리포트_20251229.md`, `docs/reports/통신/통신_연결_문제_수정_단계별_계획.md`
- 확인 코드: `src/main.py`, `src/modules/extruder.py`, `src/modules/ls_plc.py`, `src/modules/logger.py`

## 2. 반영된 항목(보고서 대비)
- 상위 루프 안전 처리: PLC 응답이 빈 딕셔너리여도 루프가 지속되도록 안전 접근 로직 추가.
- 타임아웃 상향: 압출기 타임아웃 0.15초 → 0.5~1.0초로 상향.
- 로그 경로 유효성 검증 및 로컬 fallback: 이식 PC에서 `logpath` 유효성 확인 후 실패 시 `./logs`로 자동 전환.
- 파일 기록 오류 격리/재시도: file_writer_thread 예외 발생 시 백오프/재시도와 오류 카운트로 지속 수집 보장.
- 수신 완전성 보장: 응답 길이(body_len) 또는 종결 문자열 기준으로 `recv()` 루프 구현.
- 요청 최적화(부분): 압출기 레지스터 요청을 가능한 범위에서 블록 읽기로 통합.
- 재연결/백오프 정책: 연속 실패 시 지수형 backoff로 장비 부하 감소.
- 큐/버퍼 처리량 개선(부분): Ring Buffer 용량 상향, 배치 저장, 큐 길이 모니터링으로 데이터 드롭 최소화.
- 중복 실행 방지: 단일 실행 락 및 시작 버튼 비활성화로 중복 수집 스레드 방지.

## 3. 현장 테스트 로그 확인 결과(system_2.log/CSV)
- 대상 로그: `docs/system_2.log`
- 대상 CSV: `docs/Factory_Integrated_Log_20251231_000000.csv`
- `IO Error on D0020` 시각(2025-12-31 02:15:35, 09:37:00, 09:43:04, 09:43:43, 09:57:22)과 동일 구간에서 압출기/PLC 컬럼이 31~32행 공백(약 6초)으로 기록됨.
- 09:57:22 구간은 30행이 모든 컬럼(Spot 포함) 공백으로 기록됨.
- 타임스탬프 간격은 연속이며, 행 누락은 없고 값이 빈 형태로 남음.
- `PLC Incomplete header response` 경고가 발생한 시각과 공백 구간이 동일함.

## 4. 부분 반영/차이
- 로그 경로 fallback: 문서에는 `./logs` 전환이 명시되었으나, 실제 구현은 `APP_DATA_DIR/logs` 전환으로 동작.
- 중복 실행 방지: 단일 실행 락은 반영되었으나, “시작 버튼 비활성화”는 UI 흐름상 별도 구현 없음.
- 큐/버퍼 처리량 개선: 용량 상향/모니터링 로그는 반영되었으나, 드롭 카운트/추세 지표는 미구현.

## 5. 미반영 항목
- 네트워크 품질 관측 및 장비 통신 스펙 확인: 운영 절차 항목으로 미진행.
- 성능 지표/실측 계획: 문서화된 절차 항목으로 코드 반영 없음.
