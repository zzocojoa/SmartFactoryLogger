# 전체 통신 주소 맵핑 현황

## 1. 개요

본 문서는 SmartFactoryLogger 시스템에서 수집하는 모든 데이터 항목에 대한 통신
방식, 물리적 주소(Memory Address), 그리고 데이터 처리 로직을 정리한 맵핑
테이블입니다.

## 3. 상세 설명

### 3.1 Extruder (압출기)

- **통신 프로토콜**: TCP 소켓 통신 (Melsec 프로토콜 준수)
- **데이터 처리**: 통신 효율을 위해 인접한 데이터를 블록(Block) 단위로
  읽어옵니다.
  - 예: `D0023` (압력)은 `D0020`부터 `D0039`까지 20개를 읽은 후, 4번째 값(Index
    3)을 추출하여 사용합니다.
- **주요 소스 파일**: `src/modules/extruder.py`

### 3.2 LS PLC

- **통신 프로토콜**: TCP 소켓 통신 (LS XGT 프로토콜)
- **주소 설정**: `config.ini` 파일의 `[LS_PLC_TARGETS]` 섹션에서 유동적으로
  주소를 변경할 수 있습니다.
- **단위 변환**: `At_Temp`와 `At_Pre`는 PLC에서 정수(Integer)로 수집되므로,
  100.0으로 나누어 소수점 단위로 변환합니다.
- **주요 소스 파일**: `src/modules/ls_plc.py`, `src/config.py`

### 3.3 SPOT (적외선 온도계)

- **통신 프로토콜**: HTTP GET 요청
- **URL 구조**: `http://{IP}/output?p=temperature`
- **특이 사항**: 별도의 산업용 프로토콜이 아닌 웹 요청 방식을 사용하며, 응답
  본문(Body)의 텍스트를 파싱합니다.
- **주요 소스 파일**: `src/modules/spot.py`
