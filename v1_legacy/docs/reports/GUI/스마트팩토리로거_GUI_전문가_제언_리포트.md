# 스마트팩토리로거 GUI 전문가 제언 리포트

`작성일: 2025-12-31`
`분석 대상: GUI_dashboard.png (현행 대시보드), 스마트팩토리로거_GUI_사용자_경험_개선_리포트.md`

본 리포트는 전문 디자이너의 관점에서 현재의 GUI를 분석하고, 심미성(Aesthetics)과
사용성(Usability)을 극대화하기 위한 구체적인 개선안을 제언합니다.

## 1. 총평 (Executive Summary)

현재 `customtkinter` 기반의 Dark Mode UI는 산업 현장의 시인성 요구사항을 어느
정도 충족하고 있습니다. 하지만 **"정보의 위계(Hierarchy)"**와 **"상태
인지(Status Awareness)"** 측면에서 개선 여지가 큽니다.

운영자는 0.2초마다 갱신되는 데이터의 홍수 속에서 **"지금 문제가 있는가?"**를 1초
안에 판단해야 합니다. 현재 디자인은 모든 정보가 비슷한 비중(카드 형태)으로
나열되어 있어 시선이 분산됩니다. 이를 **"맥락적 그룹핑(Contextual Grouping)"**과
**"상태 중심 컬러링(State-driven Coloring)"**으로 해결해야 합니다.

## 2. 디자인 분석 및 제언

### 2.1 레이아웃 및 정보 위계 (Layout & Hierarchy)

- **현상**: 3단 컬럼(KPI / Temp / Mold) 구조는 논리적이지만, 시각적 강약 조절이
  부족하여 평면적으로 보입니다.
- **제언**:
  1. **Hero Section 도입**: 가장 중요한 '현재 속도(Speed)'와 '메인
     압력(Press)'을 좌측 상단에 더 크게 배치하거나, 배경색을 미세하게 밝게(Card
     Highlight) 처리하여 시선을 고정시켜야 합니다.
  2. **Zone Grouping 강화**: 각 컬럼(KPI, Temp, Mold)의 헤더("⚡ PROCESS KPI"
     등)에 미세한 배경색 띠(Sub-header Background)나 컬러 바(Left Accent)를
     추가하여 영역을 명확히 구분합니다.
  3. **F-Pattern 적용**: 중요한 알림이나 상태 배지를 운영자의 시선 이동
     경로(좌상 → 우상 → 좌하)에 맞춰 배치합니다.

### 2.2 컬러 시스템 및 시인성 (Color & Visibility)

- **현상**: `InfoCard`의 타이틀과 값이 단순 텍스트로 구성되어 있음.
  `COLOR_CARD`(#333333) 단색 배경.
- **제언**:
  1. **상태 연동 배경색 (Semantic Background)**: 이상 수치 발생 시 텍스트 색상만
     빨갛게 변하는 것이 아니라, 카드 테두리(Border)나 배경색 자체가 붉은
     톤(투명도 10-20%)으로 은은하게 빛나도록(Glow Effect) 하여 주변
     시야(Peripheral Vision)에서도 인지되게 합니다.
  2. **데이터 최신성 시각화**: 데이터가 3초 이상 갱신되지 않으면(Stale), 카드
     전체의 채도를 낮추거나(Grayscale), "Paused" 오버레이를 씌워야 합니다. 값은
     그대로인데 멈춘 상태가 가장 위험합니다.
  3. **고대비 모드 지원**: 기본 Dark 테마 외에, 조명이 강한 현장을 위한 'High
     Contrast Mode'(순수 Black/White + 원색) 토글 고려.

### 2.3 상호작용 및 피드백 (Interaction & Feedback)

- **현상**: `ToastNotification`이 하단에 잠시 떴다 사라짐. 클릭 가능한
  요소(아이콘 등)의 어포던스(Affordance)가 낮음.
- **제언**:
  1. **Persistent Status Bar (P0 개선안 구체화)**:
     - 화면 최상단에 고정된 30px 높이의 상태 바를 신설합니다.
     - 좌측: 시스템 상태 (Running/Stop), 네트워크 지연 시간(ms).
     - 중앙: 최근 발생한 최상위 알림 (Rolling Text).
     - 우측: 시계, 데이터 저장 상태(Disk I/O Indicator).
  2. **Notification Drawer**: 토스트 알림을 놓친 경우를 대비해, 우측 상단
     '종(Bell)' 아이콘을 두고, 클릭 시 지나간 알림 이력(History)이 슬라이드
     다운되도록 합니다.
  3. **Tactile Feedback (Visual)**: 버튼 클릭 시 단순 색상 변경을 넘어, 미세하게
     눌리는 애니메이션(Scale Down)을 추가하여 터치 스크린에서의 조작감을
     향상시킵니다.

### 2.4 그래프 시각화 (Charts)

- **현상**: Matplotlib 기본 스타일. 범례(Legend)가 차트 영역을 가릴 수 있음.
- **제언**:
  1. **Mini-Map / Sparkline**: 대시보드의 주요 카드(InfoCard) 내부에 지난
     1분간의 추세를 보여주는 아주 작은 선 그래프(Sparkline)를 배경으로
     깔아줍니다. 이것만으로도 "상승세인지 하락세인지" 직관적 파악이 가능합니다.
  2. **Threshold Band**: 그래프 배경에 정상 범위(Normal Range)를 녹색/회색
     밴드로 칠해두어, 선이 밴드를 벗어나는 순간 시각적 긴장감을 줍니다.

## 3. 실행 로드맵 (Action Items)

| 우선순위      | 영역            | 구체적 실행 방안 (Implementation)                                                                                         |
| :------------ | :-------------- | :------------------------------------------------------------------------------------------------------------------------ |
| **즉시 (P0)** | **헤더 개편**   | `header` 프레임을 `StatusBar` 클래스로 분리. 네트워크 지연(Latency) 및 DB 저장 아이콘 추가.                               |
| **즉시 (P0)** | **카드 시인성** | `InfoCard` 클래스 수정: `update_status(level)` 메서드 추가. 정상/주의/위험에 따라 `border_color` 및 `fg_color` 동적 변경. |
| **단기 (P1)** | **스파크라인**  | `InfoCard`에 간소화된 캔버스 추가하여 최근 10개 데이터 점 연결 (Matplotlib 없이 캔버스 라인으로 경량 구현).               |
| **중기 (P2)** | **알림 센터**   | `NotificationManager` 클래스 신설. `Toast` 호출 시 이력 리스트(`deque`)에 저장하고 UI에서 열람 기능 구현.                 |

## 4. 결론

"예쁜 디자인"보다 **"말하는 디자인(Informative Design)"**이 되어야 합니다.
운영자가 화면을 쳐다보지 않고 스쳐 지나가더라도 **"어? 저기 빨간 불
들어왔네"**라고 느낄 수 있도록, **'상태에 따른 공간적 변화(Spatial Change)'**를
주는 방향으로 개선하십시오.
