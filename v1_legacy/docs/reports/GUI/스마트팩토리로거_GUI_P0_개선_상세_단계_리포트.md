# 스마트팩토리로거 GUI P0 개선 상세 단계 리포트

## 1. 목적
- P0(필수) 항목을 디자이너 실무 관점에서 단계별 실행안으로 구체화한다.
- 대상: 상단 고정 상태 바, 알림 센터, 즉시 조치 버튼.

## 2. 전제 및 설계 기준
- 공장 현장 고정형 화면, 장시간 연속 운영.
- 0.2초 수집 주기에서 지연/끊김을 즉시 인지 가능해야 한다.
- 운영자 3초 내 인지/조치 목표.

- 참고 화면: GUI_dashboard.png (현행 대시보드).
## 3. P0 상세 단계

### 3.0 전문가 제언 P0 개선안 구체
  1. **Persistent Status Bar (P0 개선안 구체화)**:
     - 화면 최상단에 고정된 30px 높이의 상태 바를 신설합니다.
     - 좌측: 시스템 상태 (Running/Stop), 네트워크 지연 시간(ms).
     - 중앙: 최근 발생한 최상위 알림 (Rolling Text).
     - 우측: 시계, 데이터 저장 상태(Disk I/O Indicator).
  2. **Notification Drawer**: 토스트 알림을 놓친 경우를 대비해, 우측 상단
     '종(Bell)' 아이콘을 두고, 클릭 시 지나간 알림 이력(History)이 슬라이드
     다운되도록 합니다.
  3. **Tactile Feedback (Visual)**: 버튼 클릭 시 단순 색상 변경을 넘어, 미세하게
     눌리는 애니메이션(Scale Down)을 추가하여 터치 스크린에서의 조작감을
     향상시킵니다.



### 3.0A 현행 화면 요소 정합화 (대시보드 기준)
- 상단 헤더(라인명/Time Series/시각/Running/설정 아이콘)와 신규 상태 바의 관계를 정의한다: 1행 헤더 + 2행 상태 바로 분리하거나, 헤더 자체를 상태 바로 확장하여 중복 요소를 제거한다.
- Running 표시는 전체 시스템 상태와 일치하도록 상태 코드를 매핑한다: Running/Degraded/Disconnected. 색상 규칙은 현행 테마(청록/주황)와 통일하고, 오류만 적색을 사용한다.
- 하단 'Last Update' 타임스탬프를 상태 바에 통합하고, 화면 하단 복제 표시를 제거한다.
- 우측 'OPERATOR CHECK' 패널은 알림 센터의 '최우선 알림 핀(Pinned Alert)'으로 연결하여 단일 소스를 유지한다.
- 상단 우측 설정(⚙)과 즉시 조치 버튼 배치를 정리한다: [재연결][진단][관리자 호출]을 상태 바 우측에 그룹화하고, 설정 아이콘은 우측 최겁진 위치를 유지한다.
- 알림 드로어는 우측 'MOLDS & ENV' 영역 위에 오버레이로 열리도록 설계해 레이아웃 흐트러지앨림을 최소화한다.
- 상태 바 높이는 28~32px 이내로 유지하고, 현행 3컬럼 카드 간격을 미세 조정해 주요 KPI/카메라 뷰가 가려지지 않게 한다.

### 3.0B 와이어프레임 레벨 배치 가이드
- 1) 상단 2행 구조: 1행 헤더(라인명/Time Series/시간/Running/설정), 2행 상태 바(연결/지연/저장/오류 요약)로 구성하여 정보 겹침을 제거한다.
- 2) 상태 바 슬롯: 좌측 2개(장치 연결, 네트워크 지연), 중앙 1개(최신 오류 요약), 우측 1~2개(NAS/저장, 즉시 조치 버튼)로 배열한다.
- 3) 알림 아이콘: 우측 상단 벨+카운트를 고정하고, 클릭 시 우측 드로어(화면 폭 28~32%)가 슬라이드로 열린다.
- 4) 좌측 KPI 컬럼: 게이지를 최상단에 배치하고, Main Pressure → Prod Count → End Position 순으로 평준 정렬한다.
- 5) 중앙 영역: SPOT Temp 카드를 가장 크게 유지하고, 카메라 뷰는 하단 중앙에 고정한다.
- 6) 우측 컬럼: Mold 카드를 상단, At Temp/At Pre를 중단, 하단은 Pinned Alert(기존 OPERATOR CHECK 위치)로 제자리하여 제한된 주목성을 확보한다.
- 7) 간격/규격: 카드 간 12~16px, 컬럼 간 16~20px, 상태 바 높이 28~32px를 유지하고 현행 3컬럼 스캔 루트를 유지한다.
- 8) 시선 유도: 좌상단(헤더/상태) → 중앙 상단(SPOT) → 우측 상단(Mold) → 우측 하단(Pinned Alert) → 중앙 하단(카메라) 순으로 사용자 스캔을 설계한다.

### 3.0C OPERATOR CHECK 문구/톤 개선
- 1) 톤 원칙: 명령형+짧은 문장, 원인/영향/조치 순으로 정리하여 즉시 행동을 유도한다.
- 2) 제목 규칙: '[조치 필요] SPOT 카메라 조준' 같이 동사 중심으로 작성하고, 심각도 상태와 연동한다.
- 3) 본문 구성: 원인/영향 1문장 + 조치 1문장로 제한하고, 핵심 단어를 색상/강조표시로 추가한다.
- 4) CTA 구성: 'SPOT 카메라 보기 열기' / '조치 완료' / '다시 알림(10분)' 3개로 정리하여 현장 흐름을 특정한다.
- 5) 예시 문구(추천): 제목='[조치 필요] SPOT 카메라 조준', 본문='제품 위치가 벗어나면 온도 측정이 실패할 수 있습니다. 십자선을 제품 중심에 맞춰 주세요.'
- 6) 상태 연동: 조치 완료 시 Pinned Alert는 축소하고 아이콘/배지로 이력만 유지한다. 미발생 시 '알림 없음' 상태로 전환한다.

### 3.0E 알림 센터 조치 문구
- 연결 끊김: 재연결 / 설정 열기 / 관리자 호출
- 데이터 지연: 상태 확인 / 재연결 / 설정 열기
- NAS 저장 실패: 설정 열기 / 재연결 / 관리자 호출


### 3.0F 미반영/추가 보완 필요 항목 (Result 반영)
- 항목: 디스크 I/O 애니메이션 (Disk Activity Indicator)
- 현상: 현재 저장 아이콘이 항상 정지 상태로 표시됨.
- 계획: 실제 데이터 기록 시 아이콘 점멸/색상 변화 피드백 기능을 추가한다.
- 조치: P1 개선 단계에서 상태 바 아이콘 연동 기준과 트리거 정의 후 구현한다.

### 3.1 상단 고정 상태 바
- 1) 정보 모델 정의: 장치 연결 상태(PLC/Extruder), 데이터 최신성(마지막 갱신/지연 초), 기록 상태(NAS/로컬), 최근 오류 요약.
- 2) 상태 기준/표현 규칙: 정상/주의/오류 3단계, 색상+아이콘+텍스트 동시 사용.
- 3) 레이아웃 설계: 상단 고정 1열, 3~5개 슬롯, 큰 글자/높은 대비.
- 4) 상호작용: 상태 바 클릭 시 알림 센터/진단 화면으로 이동, 툴팁으로 의미 설명.
- 5) 예외 상태 정의: 데이터 누락/지연 시 '지연' 배지, 연결 끊김 시 재연결 안내.
- 6) 검증: 3초 인지 테스트, 경보 누락률/오인률 측정.

### 3.2 알림 센터
- 1) 알림 구조 정의: 심각도, 시간, 대상 장치, 요약/원인, 권장 조치.
- 2) 노이즈 억제: 동일 오류 묶음, 카운트 표시, 최근 N건 유지.
- 3) 표시 방식: 우측 패널 또는 팝업, 최신 알림 상단 고정, 필터(심각도/장치).
- 4) 상태 연계: 상태 바 배지/카운트 연동, 토스트는 치명/긴급만.
- 5) 운영 흐름: 확인(ack) 처리, 상세 로그 열기, 조치 버튼 연결.
- 6) 검증: 오류 재현 시나리오로 조치 시간/정확도 측정.

### 3.3 즉시 조치 버튼
- 1) 버튼 정의: 재연결(전체/개별), 설정 열기, 관리자 호출.
- 2) 권한/위험 관리: 확인 팝업, 비활성 조건/사유 표시.
- 3) 위치/표현: 상태 바 우측 고정, 아이콘+텍스트, 터치 가능한 크기.
- 4) 피드백: 진행 상태(스피너), 완료/실패 메시지.
- 5) 실패 처리: 권한 오류/네트워크 오류 안내 문구 제공.
- 6) 검증: 오조작률, 조치 완료 시간.


### 3.4 구현 체크포인트 (전문가 제언)
| **즉시 (P0)** | **헤더 개편**   | `header` 프레임을 `StatusBar` 클래스로 분리. 네트워크 지연(Latency) 및 DB 저장 아이콘 추가.                               |
| **즉시 (P0)** | **카드 시인성** | `InfoCard` 클래스 수정: `update_status(level)` 메서드 추가. 정상/주의/위험에 따라 `border_color` 및 `fg_color` 동적 변경. |

## 4. 산출물 및 핸드오프
- 와이어프레임, 상태/알림 모델 다이어그램.
- UI 스펙(색상/타이포/간격/아이콘), 인터랙션 플로우.
- 개발 전달 체크리스트(상태 바/알림 센터/버튼 연결 기준).

## 5. 검증 방법
- 3초 인지 테스트(A/B).
- 오류/지연 시나리오 워크스루.
- 현장 운영자 3명 이상 사용성 점검.

## 6. 결론
P0 개선은 인지와 조치 속도를 높이는 구조 개선이며, 상태 바-알림 센터-즉시 조치 버튼의 연계가 핵심이다.
