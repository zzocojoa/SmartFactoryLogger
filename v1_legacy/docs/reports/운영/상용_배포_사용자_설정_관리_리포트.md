# 상용 배포 사용자 설정 관리 리포트

## 1. 목적
- Define operational rules for managing user config values in commercial deployment.
- Assume the migrated field PC uses `config.ini` as the primary source.

## 2. 현장 이식 전제
본 프로그램은 개발 환경에서 빌드 후 실제 공장 현장 PC로 이식하여 사용되는 전제를 가집니다. 따라서 `config.ini`와 `src/config.py`의 경로/모드(Portable vs Standard) 설계 및 변경은 현장 이식/운영 시나리오를 기준으로 검토하는 것이 중요합니다. 현장 PC에는 `config.ini`를 동봉 배포하여 IP/포트 변경을 운영자가 즉시 반영할 수 있도록 하는 것이 적합합니다. 이식된 PC의 `config.ini` 기준으로 실행하며, 로그 경로가 유효하지 않으면 `APP_DATA_DIR/logs`로 자동 전환합니다.

## 3. 설정 값 관리 방식
- 단일 진입점: `config.ini`만 수정, UI 변경은 파일에 저장.
- 우선순위: 이식 PC `config.ini` > APP_DATA_DIR 기본값 > 코드 기본값.
- 검증: `config_schema.py`가 타입/범위 검증, 누락 값은 기본값 보정.
- 안전 저장: 임시 파일 작성 후 원자적 교체, `config.ini.bak` 순환 백업 유지.
- 자동 전환: 로그/스냅샷 경로가 유효하지 않으면 `APP_DATA_DIR`로 전환, 경고 로그 기록.
- 보안: 가능하면 평문 설정에 인증 정보 저장 금지.

## 4. 보완 제안 (Advanced Tips)
- 인코딩: 한국어 윈도우 메모장 대응을 위해 `config.ini`를 UTF-8 with BOM으로 생성하고, 로딩 시 `utf-8-sig` 우선 + 실패 시 `cp949` fallback을 둔다.
- 환경 변수: 자동 배포 대비로 환경 변수가 있으면 `config.ini`보다 우선하도록 하고(예: `MYAPP_SERVER_IP`), 덮어쓴 값은 로그로 남긴다.
- 읽기 전용: `PermissionError` 발생 시 “관리자 권한이 필요합니다” 같은 명확한 UI 안내를 제공하고, 저장 실패 시 임시 경로 저장/재시도 정책을 둔다.

## 5. 구현 skills
- 설정 스키마 설계 및 검증(Pydantic).
- 원자적 파일 I/O 및 백업/롤백 전략.
- 환경 감지(Portable/Standard)와 경로 설계.
- 설정 실패에 대한 운영 로그/감시.
- 배포 패키징 및 롤백 절차.

## 6. 검증 방법
- 이식 PC `config.ini` 수정 후 반영 확인.
- 잘못된 타입/범위 주입 후 기본값 보정과 경고 로그 확인.
- 읽기 전용 경로에서 자동 전환 및 데이터 손실 여부 확인.
- 동시 실행 시 `app.lock`으로 설정 파일 무결성 확인.

## 7. 결론
- 상용 배포는 설정 검증, 안전 저장, 우선순위 규칙이 필수.
- 현장 PC의 `config.ini`가 사용자 설정의 단일 기준.
